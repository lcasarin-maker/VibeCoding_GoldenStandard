### Symptom (Signal)
> Governance files (AGENT.md, STATUS.md, SPEC.md) grow without control, saturating the context window in every session

### Cause
Absence of defined and audited physical thresholds for the core Trinity documents

### Application / Mitigation
Define and enforce line limits in each manifest (e.g. AGENT.md <= 150 lines per Trinity); audit them as a commit gate in the governance validator

### Operational Relevance
Core memory manifests must stay under fixed line budgets so context is not saturated; a static size gate fails when a manifest exceeds its budget.
