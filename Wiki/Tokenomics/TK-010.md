# TK-010: Bloated tool schemas

| **Severity** | **low** |
| **Depth** | 🟢 Deep |

| **Downstream Verification** | `none` |
| **Validation Mechanism** | `runtime-test` |

---

### Symptom (Signal)
> Unused tools consume context

### Cause
Excessive upfront loading

### Application / Mitigation
Deferred tool loading

### Operational Relevance
Load tools on demand so unused schemas do not consume context; the canonical input-and-retrieval guidance must stay published and linked from the index, which a CI link-check verifies.


### ❌ Example of the vice (Bad)
```python
tools = load_all_tools()        # schemas of 40 unused tools occupy context
```

### ✅ Corrected version (Good)
```python
tools = load_tools_for(task)    # deferred loading as needed
```

### 🔎 Concrete detection
Loading schemas of unused tools into context instead of deferred loading.

### 📚 External evidence
- **vexp.dev / HumanLayer ACE** — unstructured context consumes 3-5x more tokens; selective retrieval, caching, and compaction are the central levers.
