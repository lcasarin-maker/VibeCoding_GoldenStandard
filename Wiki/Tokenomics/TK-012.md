### Symptom (Signal)
> Many tokens before acting

### Cause
Blind exploration

### Application / Mitigation
Use indexes, maps, and targeted reading

### Operational Relevance
Input-retrieval topology validation now keeps the reading paths and map surfaces canonical instead of free-form exploration.

### ❌ Example of the vice (Bad)
```bash
cat src/**/*.py | head -3000   # many tokens before acting
```

### ✅ Corrected version (Good)
```bash
ctags -R src/ && grep -n 'def target' tags   # index + targeted reading
```

### 🔎 Concrete detection
Spending many tokens exploring blindly before acting, without indexes or maps.

### 📚 External evidence
- **vexp.dev / HumanLayer ACE** — unstructured context consumes 3-5x more tokens; selective retrieval, caching, and compaction are the central levers.
