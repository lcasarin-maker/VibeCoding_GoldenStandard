### Symptom (Signal)
> Switching threads loses state

### Cause
Non-external persistence

### Application / Mitigation
Keep state outside the chat

### Operational Relevance
Keep authoritative state outside the chat so switching threads does not lose it; the canonical memory-and-headroom guidance must stay published and linked from the index, which a CI link-check verifies.


### ❌ Example of the vice (Bad)
```text
# State lives in the chat thread; switching threads loses it.
```

### ✅ Corrected version (Good)
```text
# State outside the chat (STATUS.md) as the primary source.
```

### 🔎 Concrete detection
Using chat memory as the source of truth for state instead of an external artifact.

### 📚 External evidence
- **vexp.dev / HumanLayer ACE** — unstructured context consumes 3-5x more tokens; selective retrieval, caching, and compaction are the central levers.
