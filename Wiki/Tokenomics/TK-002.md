### Síntoma (Signal)
> Cambiar hilo pierde estado

### Causa (Cause)
Persistencia no externa

### Aplicación / Mitigación
Mantener estado fuera del chat

### Relevancia Operativa
Tokenomics topology validation keeps the canonical memory surfaces visible, so chat is not treated as the only durable store.

### ❌ Ejemplo del vicio (Bad)
```text
# El estado vive en el hilo de chat; cambiar de hilo lo pierde.
```

### ✅ Versión corregida (Good)
```text
# Estado fuera del chat (STATUS.md) como fuente principal.
```

### 🔎 Detección concreta
Usar la memoria del chat como fuente de verdad del estado en vez de un artefacto externo.

### 📚 Evidencia externa
- **vexp.dev / HumanLayer ACE** — el contexto sin estructura consume 3-5x mas tokens; recuperacion selectiva, caching y compactacion son las palancas centrales.
