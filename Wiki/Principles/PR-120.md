## Doctrine Snapshot

| Field | Value |
|---|---|
| **Symptom** | Agent writes hundreds of lines in one turn, skips cleanup, and accumulates a growing surface that cannot be reviewed or reverted atomically. |
| **Cause** | No turn-level output constraint; generative bias toward completeness over minimalism. |
| **Solution** | Hard cap of 50 lines per turn. After each change: count lines added, run Simplicity Pass, commit only what passes both checks. |
| **Action** | Limit net new code to 50 lines per conversation turn. After each change, run a Simplicity Pass: ask what can be deleted before declaring the change complete. Large-batch writes accumulate unreviewed debt faster than it can be absorbed. |
