## Doctrine Snapshot

| Field | Value |
|---|---|
| **Symptom** | Intentional simplification (O(n²) loop, in-memory cache, hardcoded limit) with no comment naming when it is safe and when it must change. |
| **Cause** | Agent takes shortcut to ship, intends to revisit, leaves no machine-readable marker. Shortcut becomes permanent because there is no ledger. |
| **Solution** | Comment form: # ponytail: <ceiling>, <upgrade trigger> Example: # ponytail: global lock, upgrade when concurrent writes exceed 1 Run ponytail-debt (grep ponytail:) periodically. Any ponytail: comment without a trigger clause is flagged no-trigger — highest rot risk. |
| **Action** | When taking a deliberate shortcut (global lock, O(n²) scan, naive heuristic, single-threaded assumption), mark it with a ponytail: comment naming the ceiling (what breaks first) and the upgrade path (what triggers the revisit). A shortcut without a named ceiling is silent debt. |
