## Doctrine Snapshot

| Field | Value |
|---|---|
| **Symptom** | An audit framework has PASS and FAIL but no explicit CNV state; checks that cannot run silently appear as PASS, giving false confidence. |
| **Cause** | Binary thinking: the audit author assumes that running and finding nothing is the only alternative to finding a problem. |
| **Solution** | Add a third mandatory status (CNV) that surfaces when a check cannot be completed. Treat CNV as neither PASS nor FAIL at the gate, but never hide it. |
| **Action** | When implementing any audit check, define the condition under which it cannot be verified and emit CNV with a reason string when that condition holds. Audit frameworks must expose all three states. |
