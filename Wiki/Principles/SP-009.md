## Doctrine

| Field | Value |
|---|---|
| **Symptom** | Decisions, rationale, and session findings live only in chat or agent memory; next session has no durable record and repeats the same discoveries or makes contradictory decisions without knowing why the prior choice was made. |
| **Cause** | Agents treat canonicalization as optional clean-up rather than a required gate; same failure mode that GS's INGESTION_PROTOCOL exists to prevent in the knowledge base. |
| **Solution** | Before session close, every material decision is written to DECISIONS.md (durable why), every session finding to audit/sessions/ (durable what happened), and every new work item to tasks/backlog/ (durable what remains). Nothing that affects the project may exist only in chat. |
| **Action** | Treat DECISIONS.md + audit/sessions/ + tasks/backlog/ as the project's ingestion pipeline: same quality gate as GS INGESTION_PROTOCOL, applied to project knowledge instead of coding-vice knowledge. |

---

## Detection

audit.py: assert audit/sessions/ has a record for current session; assert DECISIONS.md was updated if any architectural choice was made; assert tasks/backlog/ contains any newly discovered work items
