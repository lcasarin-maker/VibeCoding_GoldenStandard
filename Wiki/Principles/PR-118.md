## Doctrine Snapshot

| Field | Value |
|---|---|
| **Symptom** | Agent uses shell file-mutation commands to modify files, making changes unauditable and rollback-unsafe. |
| **Cause** | Default shell familiarity; no explicit prohibition in base model behavior. |
| **Solution** | Use Edit for targeted replacements under 50 lines; use Write for full file creation. Never use Bash/PowerShell shell operations to write file content. |
| **Action** | All file writes must use atomic Edit/Write tools. Shell file-mutation operations (write-redirect, in-place sed, PowerShell file-append cmdlets) are prohibited because they bypass line-level audit, can corrupt files silently, and cannot be reviewed atomically. If a write primitive is unavailable, escalate instead of falling back to shell. |
