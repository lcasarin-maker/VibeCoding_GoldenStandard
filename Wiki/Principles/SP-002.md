## Doctrine

| Field | Value |
|---|---|
| **Symptom** | Decisions, blockers, rationale, or task state exist only in chat transcript and are lost on session end |
| **Cause** | Convenience; agent treats chat as durable storage |
| **Solution** | Any fact affecting project state is written into project files before session closes (HANDOFF, STATE, DECISIONS, or tasks/) |
| **Action** | Enforce mandatory session-close phase; HANDOFF.md and audit/sessions/ record are required outputs of every session. |

---

## Detection

audit.py: assert audit/sessions/ contains a record dated today; assert HANDOFF.md mtime matches today
