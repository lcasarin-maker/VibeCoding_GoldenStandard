## Doctrine Snapshot

| Field | Value |
|---|---|
| **Symptom** | Agent declares success based on code that looks correct, without running it. Test output, CI, or runtime evidence is absent from the success claim. |
| **Cause** | LLM generative bias toward positive closure; no explicit empirical gate in default behavior. |
| **Solution** | S0 pre-success checklist in AUDIT_TRAIL.md. B7 prohibits declaring success without real logs or human confirmation. Default posture: assume failure until the gate is passed. |
| **Action** | Before declaring a task complete, verify empirically: run the test suite, read the actual output, check the audit log. Declaring success without terminal evidence is optimism theater. Checklist required in AUDIT_TRAIL.md before each success claim. |
