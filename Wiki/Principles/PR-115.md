# PR-115 — External Tool Attribution: every evaluated tool must be cataloged with source, license, and verdict before adoption or rejection

When an agent evaluates, installs, or recommends an external tool, library, or project, that evaluation must be recorded in the project's external tools catalog (`knowledge/external_tools.md`) with: the canonical source URL, license, verdict (adopted/backlog/rejected), and the rationale. Credit is non-negotiable — omitting attribution to the original authors is a documentation debt and a reproducibility failure.

Origin: Session 2026-06-23 — tools analyzed (terser, token-optimizer, python-minifier, headroom-ai, LLMLingua) had no formal attribution record in the project.

## Doctrine Snapshot

| Field | Value |
|---|---|
| **Symptom** | Tools are evaluated in session and referenced in backlog tasks but never formally attributed; future agents cannot verify source or license |
| **Cause** | No catalog obligation — attribution is treated as optional documentation |
| **Solution** | Maintain `knowledge/external_tools.md` as the single source of truth for all evaluated external tools; update it before closing any task that involves an external dependency |
| **Action** | Before adopting or backlisting any external tool: add an entry to `knowledge/external_tools.md` with URL, license, verdict, and origin session. Backlog tasks must reference the entry. |
