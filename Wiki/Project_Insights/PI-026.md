# PI-026: Insight Satélite

### Descripción del Insight
> Preflight exhaustivo y no reabrir alcance post-ejecución – antes de ejecutar cualquier cambio, el agente debe declarar scope, impactos previsibles, follow-ups fuera de alcance y el runner/loader que se vería afectado si la topología cambia. Si después de ejecutar surge una mejora nueva, se registra primero en backlog; no se ofrece como sugerencia post-ejecución. Origen: Cerberus / GS execution hygiene y la separación formal de auditorías entre conocimiento y consumo.
