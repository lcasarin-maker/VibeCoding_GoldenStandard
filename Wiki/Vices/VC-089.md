### Symptom
> Agent adds new code, files, or parameters to solve a problem that removing existing code would have solved

### Cause
Generative bias — LLMs are trained to produce tokens, so addition is the default action even when the correct answer is a diff that shrinks

### Solution
Before writing any new code, ask whether removing or simplifying existing code solves the problem. Deletion is cheaper, safer, and leaves no new surface to maintain.

### Corrective Action / Prevention
Run the YAGNI check before writing: (1) does this need to exist at all? (2) does removing something existing solve it? Shortest working diff wins. Deletion over addition. Flag any PR where net line count is positive but the root problem was an unused code path.


### ❌ Example of the vice (Bad)
```python
# Bug: function raises on empty list. Agent adds a guard wrapper.
def safe_first(lst):
    if not lst: return None
    return first(lst)   # new function wrapping existing one
```

### ✅ Corrected version (Good)
```python
# Fix: the existing function already has the right shape — just call it with a default.
first(lst, default=None)   # stdlib/existing API; no new function needed
```

### 🔎 Concrete detection
Heuristic: PR adds >10 lines to fix a bug that a 1-line deletion or stdlib call would resolve. Review tag: delete: — what to cut, replacement is nothing or a stdlib call.
