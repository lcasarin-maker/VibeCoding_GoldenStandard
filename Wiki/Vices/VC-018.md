### Symptom
> A temporary simulation becomes a dependency

### Cause
Substitute confused with component

### Solution
Remove stubs before validating integration

### Corrective Action / Prevention
Static detector `vc061_constant_stub` flags production functions or classes whose body collapses to a stub such as pass, ellipsis, or a constant return.

### ❌ Example of the vice (Bad)
```python
# A temporary stub stays wired as a real production dependency.
class PaymentGateway:
    def charge(self, amount):
        return True        # stub: "we'll connect it later"
```

### ✅ Corrected version (Good)
```python
# The stub lives only in tests; production uses the real implementation.
class PaymentGateway:
    def charge(self, amount):
        return self._provider.charge(amount)
```

### 🔎 Concrete detection
Classes/functions with a trivial body (return True/None/pass) imported from production
modules (not from tests/fixtures).
