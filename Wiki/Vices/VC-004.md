### Symptom
> Success declarations based solely on superficial aesthetic similarity

### Cause
Lack of empirical validation at runtime or persistence

### Solution
Subordinate approval to real logical traces or observable storage deltas

### Corrective Action / Prevention
Empirical proof is required before declaring success: closure fails when no structured evidence artifact backs the claim, forcing verification over optimism.


### ❌ Example of the vice (Bad)
```python
# 'Already fixed' asserted in the chat without running anything.
# The code ships with a triumphalist comment instead of a test.
def parse(x):
    return int(x)  # "fix: now handles all cases"
```

### ✅ Corrected version (Good)
```python
# The fix is backed by a test that exercises the claimed case.
def parse(x):
    return int(x.strip())

def test_parse_trims():
    assert parse("  42 ") == 42  # evidence, not narrative
```

### 🔎 Concrete detection
Review heuristic: reject PRs whose only support for 'it works' is prose
(comments/commit message) without a new test covering the asserted case.
