### Síntoma (Symptom)
> Pruebas exitosas inútiles que no discriminan rupturas lógicas

### Causa (Cause)
Gamificación de cobertura priorizando métrica sintáctica sobre rigurosaidad

### Solución (Solution)
Exigir aserciones de valores límite y prohibir capturas globales que silencien fallos

### Acción Correctiva / Prevención
Checked by D9 (Test Purity) using AST TestTheaterVisitor to flag assert True, assertTrue(True), assertEqual(x, x), and tests without active asserts.

### ❌ Ejemplo del vicio (Bad)
```python
# 'Verde' garantizado: combinacion de asserts tautologicos.
def test_calc():
    assert 1 == 1
    assert isinstance(calc(2), int)   # no fija el valor
```

### ✅ Versión corregida (Good)
```python
def test_calc():
    assert calc(2) == 4
```

### 🔎 Detección concreta
AST: tests cuyos asserts son todos tautologicos o de tipo (isinstance) sin fijar
ningun valor esperado concreto.

### 📚 Evidencia externa
- **arxiv:2602.00409** — estudio empirico confirma que los agentes generan tests sin oraculo (sin assert, tautologicos o sobre-mockeados) de forma medible.
