### Symptom
> Invented dependencies

### Cause
External world assumed

### Solution
Verify real integrations

### Corrective Action / Prevention
The agent imports or references a package, API, or module that does not exist
in the declared dependency manifest or in the public package registry.


### ❌ Example of the vice (Bad)
```python
# Importing a package that does not exist in requirements.txt or PyPI.
import foobar_ai_sync  # hallucinated package name
result = foobar_ai_sync.process(data)
```

### ✅ Corrected version (Good)
```python
# Only import packages that are declared in the dependency manifest.
import requests  # listed in requirements.txt
response = requests.get(url)
```

### 🔎 Concrete detection
AST: parse top-level imports and compare them against the dependency manifest
(requirements.txt, pyproject.toml, package-lock.json). Flag imports that resolve
to packages absent from the manifest. Also: check PyPI existence for new imports.
