### Síntoma (Symptom)
> El test pasa aunque el entorno real no puede iniciar

### Causa (Cause)
Precondiciones fuera del oráculo

### Solución (Solution)
Probar arranque, dependencias y permisos mínimos

### Acción Correctiva / Prevención
Checked by setup_validate.py which runs comprehensive pre-flight verification of Python, git hooks, write access, encoding, and the project registry.

### ❌ Ejemplo del vicio (Bad)
```python
# Asume que el fixture cargo bien; si esta vacio, el test 'pasa' sin probar nada.
def test_rows():
    rows = load_fixture("data.csv")
    for r in rows:                 # si rows == [] el bucle no corre
        assert r.valid
```

### ✅ Versión corregida (Good)
```python
def test_rows():
    rows = load_fixture("data.csv")
    assert len(rows) == 3          # valida el setup antes de iterar
    assert all(r.valid for r in rows)
```

### 🔎 Detección concreta
Bucles 'for ... assert' dentro de tests sin una asercion previa de no-vacuidad del
conjunto iterado (assert len > 0).
