### Symptom
> The test passes even though the real environment cannot start

### Cause
Preconditions outside the oracle

### Solution
Test startup, dependencies, and minimum permissions

### Corrective Action / Prevention
Validate that the real environment can start - runtime, dependencies, and minimum permissions - not just that the test passes.


### ❌ Example of the vice (Bad)
```python
# Assumes the fixture loaded fine; if empty, the test 'passes' without testing anything.
def test_rows():
    rows = load_fixture("data.csv")
    for r in rows:                 # if rows == [] the loop does not run
        assert r.valid
```

### ✅ Corrected version (Good)
```python
def test_rows():
    rows = load_fixture("data.csv")
    assert len(rows) == 3          # validate the setup before iterating
    assert all(r.valid for r in rows)
```

### 🔎 Concrete detection
'for ... assert' loops inside tests without a prior non-emptiness assertion of the
iterated set (assert len > 0).
