### Symptom
> An `ImportError` triggers `pip install` without human review

### Cause
The error handler delegates supply-chain decisions to the environment without control

### Solution
Forbid automatic `pip install`; report the missing dependency with manual installation guidance and stop; the human decides what gets installed

### Corrective Action / Prevention
Dependency installation must never be automatic: an import-time guard must not trigger package installation; missing packages are surfaced for manual decision.


### ❌ Example of the vice (Bad)
```python
# Runtime dependency installation, without pinning or verifying.
import os
os.system("pip install some-pkg")   # arbitrary version, unaudited code
import some_pkg
```

### ✅ Corrected version (Good)
```python
# Dependency declared and pinned in the manifest; zero runtime installation.
# pyproject.toml: dependencies = ["some-pkg==1.4.2"]
import some_pkg
```

### 🔎 Concrete detection
Detect pip/npm/installer invocations inside application code
(os.system, subprocess) instead of the dependency manifest.
