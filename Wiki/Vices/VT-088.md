### Síntoma (Symptom)
> El sistema acepta error conocido

### Causa (Cause)
Política de excepción infinita

### Solución (Solution)
Cero errores aceptados en veredicto de salud

### Acción Correctiva / Prevención
Enforced by D5 (Angry Path) AST TryBlockVisitor flagging empty try-except blocks or silent pass/continue statements.

### ❌ Ejemplo del vicio (Bad)
```python
# El test tolera errores: cualquier resultado pasa.
def test_sync():
    result = sync()
    assert result is not None or result is None   # siempre cierto
```

### ✅ Versión corregida (Good)
```python
def test_sync():
    assert sync().synced_count == 10
```

### 🔎 Detección concreta
AST: asserts cuya condicion es una tautologia logica (X or not X) o que aceptan
cualquier valor del SUT.

### 📚 Evidencia externa
- **arxiv:2602.00409** — estudio empirico confirma que los agentes generan tests sin oraculo (sin assert, tautologicos o sobre-mockeados) de forma medible.
