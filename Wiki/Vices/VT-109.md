### Symptom
> Using complex test frameworks (`pytest`) for simple static/structural validations that can be resolved with a direct, standalone script. It adds dependencies and duplication, and creates compatibility bridges that violate the principles of simplicity and minimal coupling (KISS).

### Cause
Technical inertia of using traditional development tools instead of unifying authority in a direct, sovereign auditor.

### Solution
Remove the intermediary bridge; the static auditor runs directly and natively (`exit 0` or `exit 1`).

### Corrective Action / Prevention
Testing Bridge Theater is bypassed; static audits and tests run as direct shell pipelines returning native exit codes.

### ❌ Example of the vice (Bad)
```python
# Framework theater: layers of mocks that test the mock, not the system.
def test_service():
    svc = Mock()
    svc.process.return_value = "ok"
    assert svc.process() == "ok"   # asserts against the mock itself
```

### ✅ Corrected version (Good)
```python
def test_service():
    svc = RealService(repo=InMemoryRepo())
    assert svc.process(order).state == "ok"   # real system
```

### 🔎 Concrete detection
Detect tests where the asserted object is a Mock configured in the test itself
(assert on a preprogrammed return_value).
