### Symptom
> Declared state does not match the system

### Cause
Persistence of unrevalidated claims

### Solution
Subordinate documentation to execution

### Corrective Action / Prevention
Documentation must not lie: link-integrity and count-parity validation fail when docs drift away from the generated state.


### ❌ Example of the vice (Bad)
```text
# README says "supports OAuth" but the code does not implement it.
```

### ✅ Corrected version (Good)
```text
# Doc subordinated to execution: a test verifies each claim declared in the README.
```

### 🔎 Concrete detection
Capability claims in docs without code/test backing them (doc-code divergence).

- **arxiv:2603.28592** — recurring anti-pattern of AI-generated code, present in 80-100% of the analyzed projects.
