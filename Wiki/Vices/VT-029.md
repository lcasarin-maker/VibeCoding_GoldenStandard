### Symptom
> Does not test permissions or locks

### Cause
Idealized resource

### Solution
Tests with a real resource

### Corrective Action / Prevention
Use a real resource for permission- and lock-sensitive paths that a fake filesystem cannot reproduce.


### ❌ Example of the vice (Bad)
```python
fs = FakeFS()   # never raises PermissionError or ENOSPC
```

### ✅ Corrected version (Good)
```python
# use a real temporary directory: IO with real errors
```

### 🔎 Concrete detection
Fake FS that does not reproduce real IO errors (permissions, space).
