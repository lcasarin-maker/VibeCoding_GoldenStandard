### Síntoma (Symptom)
> Archivos de protocolo copiados a proyectos hijo sin confirmar que el protocolo ejecuta

### Causa (Cause)
Confundir distribución con adopción operacional

### Solución (Solution)
Verificar post-copia que el destinatario tiene hooks, auditor y suite activos; reportar % adopción real

### Acción Correctiva / Prevención
Behavioral/doctrinal vice — not statically falsifiable in a generic way. Documented in the Golden Standard catalogs as governance knowledge; no automated test can discriminate this without human semantic judgment. Sprint 3.4 triage: reclassified from AUDITED/test_behavioral_compliance to DOC_ONLY.

### ❌ Ejemplo del vicio (Bad)
```bash
cp .protocol/* child-repo/      # copiado, pero nadie verifica que ejecute
```

### ✅ Versión corregida (Good)
```bash
cp .protocol/* child-repo/
(cd child-repo && verify_adoption)   # confirma hooks + auditor + suite activos; reporta % real
```

### 🔎 Detección concreta
Copiar archivos de protocolo a proyectos hijo sin un check post-copia que confirme
hooks/auditor/suite activos y reporte adopcion real.
