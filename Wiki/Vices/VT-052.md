### Síntoma (Symptom)
> Pipeline ignora fallo

### Causa (Cause)
Política permisiva

### Solución (Solution)
Propagación estricta

### Acción Correctiva / Prevención
Enforced by D5 (Angry Path) AST TryBlockVisitor flagging empty try-except blocks or silent pass/continue statements.

### ❌ Ejemplo del vicio (Bad)
```yaml
# CI configurado para ignorar errores: el rojo se vuelve verde artificialmente.
- run: pytest
  continue-on-error: true
```

### ✅ Versión corregida (Good)
```yaml
- run: pytest          # un fallo de tests detiene el pipeline
```

### 🔎 Detección concreta
Buscar 'continue-on-error: true', '|| true', o '--exitfirst' invertido en pasos de
test del CI que enmascaran fallos.

### 📚 Evidencia externa
- **arxiv:2602.00409** — estudio empirico confirma que los agentes generan tests sin oraculo (sin assert, tautologicos o sobre-mockeados) de forma medible.
