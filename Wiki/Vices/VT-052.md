### Symptom
> The pipeline ignores the failure

### Cause
Permissive policy

### Solution
Strict propagation

### Corrective Action / Prevention
Propagate errors strictly through the pipeline rather than ignoring them.


### ❌ Example of the vice (Bad)
```yaml
# CI configured to ignore errors: red turns green artificially.
- run: pytest
  continue-on-error: true
```

### ✅ Corrected version (Good)
```yaml
- run: pytest          # a test failure stops the pipeline
```

### 🔎 Concrete detection
Look for 'continue-on-error: true', '|| true', or an inverted '--exitfirst' in CI
test steps that mask failures.
