### Síntoma (Symptom)
> Cobertura gamificada

### Causa (Cause)
Incentivo equivocado

### Solución (Solution)
Calidad de aserciones

### Acción Correctiva / Prevención
Checked by D9 (Test Purity) using AST TestTheaterVisitor to flag assert True, assertTrue(True), assertEqual(x, x), and tests without active asserts.

### ❌ Ejemplo del vicio (Bad)
```python
# Tests vacios creados solo para alcanzar el umbral de % de cobertura.
def test_a(): import mod          # 'cubre' el import
def test_b(): mod.helper()        # llama sin verificar
```

### ✅ Versión corregida (Good)
```python
# La meta es comportamiento verificado, no el numero de cobertura.
def test_helper_doubles():
    assert mod.helper(2) == 4
```

### 🔎 Detección concreta
Detectar tests cuyo proposito declarado/commit es 'subir coverage' y cuyo cuerpo
carece de asserts sobre resultados.

### 📚 Evidencia externa
- **arxiv:2602.00409** — estudio empirico confirma que los agentes generan tests sin oraculo (sin assert, tautologicos o sobre-mockeados) de forma medible.
