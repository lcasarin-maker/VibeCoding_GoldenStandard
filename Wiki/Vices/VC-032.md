### Symptom
> Warnings accumulate without blocking; filterwarnings("ignore") without justification

### Cause
Accepted gradual degradation; blind global suppression of alert signals

### Solution
Zero warnings on health paths; every warning suppression must have a justifying comment

### Corrective Action / Prevention
Static source analysis flags a blanket `filterwarnings("ignore")` that carries no
justifying inline comment on production paths (tests excluded). The allowed pattern
is a narrowly-scoped suppression accompanied by a specific reason comment; the
rejected signature is the unqualified global suppression call.


### ❌ Example of the vice (Bad)
```python
warnings.filterwarnings("ignore")   # all are silenced, without justification
```

### ✅ Corrected version (Good)
```python
warnings.filterwarnings("ignore", category=DeprecationWarning)  # only this one, see BUG-30
```

### 🔎 Concrete detection
Global filterwarnings("ignore") or accumulation of warnings without justification on health paths.
