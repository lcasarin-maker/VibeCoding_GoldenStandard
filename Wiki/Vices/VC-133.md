### Síntoma (Symptom)
> Una llamada a herramienta falla (timeout, error, salida inesperada) y el agente continua como si hubiera tenido exito, o entra en un bucle de reintentos sin backoff.


### Causa (Cause)
No se valida el resultado de cada tool call; ausencia de manejo de error y de politica de backoff/limite en el bucle del agente.


### Solución (Solution)
Validar cada resultado de tool; aplicar backoff y limite de reintentos; degradar explicitamente ante fallo persistente.


### Acción Correctiva / Prevención
Documentado; enforcement downstream. Receta: verificar exito de cada tool call, backoff con limite, y abortar/degradar tras N fallos (ver VC-120).


### ❌ Ejemplo del vicio (Bad)
```python
result = call_tool("read_file", path)   # devolvio un error
proceed(result)                          # se usa el error como si fuera contenido
```

### ✅ Versión corregida (Good)
```python
result = call_tool("read_file", path)
if not result.ok:
    result = retry_with_backoff(max=3) or degrade()   # backoff + limite + degradacion
```

### 🔎 Detección concreta
Resultados de tool calls usados sin verificar exito; bucles de reintento sin backoff ni limite. Relacionado con VC-120 (runaway loops) y VC-058 (sin timeout).
