### Symptom
> The agent detects technical debt or a finding, verbally classifies it as "postpone / isolated sprint / deferred / later", and closes the response without opening the Sprint Document (e.g., PLAN.md). The item disappears.

### Cause
A context switch between analysis mode and implementation mode disables the VC-048 trigger; the agent assumes that "naming = registering".

### Solution
Mandatory B8 trigger: every "deferred" classification triggers immediate opening of the Debt Registry (e.g., BACKLOG.md) to register the ID + evidence, and the Sprint Document (e.g., PLAN.md) to design the implementation. No ID in the Debt Registry (e.g., BACKLOG.md) = forgotten, not deferred. (Lesson: omitting the formal registration of structural exclusions or technical debt in the centralized backlog leads to persistent drift and loss of quality control).

### Corrective Action / Prevention
Every deferred classification must immediately open a debt-registry entry; a verbal deferral with no record loses the finding.

### 🔎 Concrete detection
Classifying something as deferred without registering it as a trackable item.
