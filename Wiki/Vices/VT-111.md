### Symptom
> The agent detects technical debt or a finding, verbally classifies it as "postpone / isolated sprint / deferred / later", and closes the response without opening `PLAN.md`. The item disappears.

### Cause
A context switch between analysis mode and implementation mode disables the VC-114 trigger; the agent assumes that "naming = registering".

### Solution
Mandatory B8 trigger: every "deferred" classification triggers immediate opening of BACKLOG.md to register the ID + evidence, and PLAN.md to design the implementation. No ID in BACKLOG.md = forgotten, not deferred. (Lesson: omitting the formal registration of structural exclusions or technical debt in the centralized backlog leads to persistent drift and loss of quality control).

### Corrective Action / Prevention
Audited by DeepForensicAuditor D8 and D9 behavioral and static test validations.

### ❌ Example of the vice (Bad)
```text
# Something is marked 'deferred' without registering an actionable item.
```

### ✅ Corrected version (Good)
```text
# every 'deferred' opens an issue with a trackable ID
```

### 🔎 Concrete detection
Classifying something as deferred without registering it as a trackable item.
