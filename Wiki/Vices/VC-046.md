### Symptom
> Protocol files copied to child projects without confirming that the protocol executes

### Cause
Confusing distribution with operational adoption

### Solution
Verify post-copy that the recipient has active hooks, auditor, and suite; report real adoption %

### Corrective Action / Prevention
Behavioral/doctrinal vice — not statically falsifiable in a generic way. Documented in the Golden Standard catalogs as governance knowledge; no automated test can discriminate this without human semantic judgment. Sprint 3.4 triage: reclassified from AUDITED/test_behavioral_compliance to DOC_ONLY.

### ❌ Example of the vice (Bad)
```bash
cp .protocol/* child-repo/      # copied, but no one verifies it executes
```

### ✅ Corrected version (Good)
```bash
cp .protocol/* child-repo/
(cd child-repo && verify_adoption)   # confirms active hooks + auditor + suite; reports real %
```

### 🔎 Concrete detection
Copying protocol files to child projects without a post-copy check that confirms
active hooks/auditor/suite and reports real adoption.
