### Síntoma (Symptom)
> Lo temporal queda permanente

### Causa (Cause)
Falta de frontera de expiración

### Solución (Solution)
Todo prototipo debe tener fecha, dueño y salida

### Acción Correctiva / Prevención
Behavioral/doctrinal vice — not statically falsifiable in a generic way. Documented in the Golden Standard catalogs as governance knowledge; no automated test can discriminate this without human semantic judgment. Sprint 3.4 triage: reclassified from AUDITED/test_behavioral_compliance to DOC_ONLY.

### ❌ Ejemplo del vicio (Bad)
```python
# Prototipo "temporal" sin fecha ni dueno: se queda para siempre.
def quick_hack():            # TODO: temporal, quitar
    return cached_guess()
```

### ✅ Versión corregida (Good)
```python
# Todo prototipo lleva expiracion y dueno explicitos y rastreables.
def quick_hack():            # PROTOTYPE owner=luis expires=2026-07-01 ticket=BUG-12
    return cached_guess()
```

### 🔎 Detección concreta
Marcar comentarios TODO/FIXME/HACK/'temporal'/'prototype' sin dueno + fecha de
expiracion + ticket parseables.
