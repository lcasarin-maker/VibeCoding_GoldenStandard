### Symptom
> Secondary failures stay alive

### Cause
Poorly defined closure scope

### Solution
Close the full chain of revealed defects

### Corrective Action / Prevention
Closure is all-or-nothing: the catalog validator keeps canonical surfaces, link graph, and coverage counts aligned, so a partial close fails the gate.


### ❌ Example of the vice (Bad)
```python
def fix():
    patch_login()        # fixes the main bug
    # the log also reveals a secondary KeyError -> ignored and closed
```

### ✅ Corrected version (Good)
```python
def fix():
    patch_login()
    assert not scan_logs().errors   # closes the full chain of defects
```

### 🔎 Concrete detection
Closing a task while the logs/CI still show secondary errors revealed by the fix itself.
