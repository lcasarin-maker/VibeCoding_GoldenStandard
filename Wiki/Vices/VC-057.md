### Symptom
> Docs, runners, and state declare different versions

### Cause
Inconsistent distributed identity

### Solution
One canonical version, derived not copied

### Corrective Action / Prevention
Behavioral/doctrinal vice — not statically falsifiable in a generic way. Documented in the Golden Standard catalogs as governance knowledge; no automated test can discriminate this without human semantic judgment. Sprint 3.4 triage: reclassified from AUDITED/test_behavioral_compliance to DOC_ONLY.

### ❌ Example of the vice (Bad)
```text
# Three sources declare different versions.
pyproject.toml : version = "1.4.0"
__init__.py    : __version__ = "1.2.0"
CHANGELOG.md   : ## 1.5.0
```

### ✅ Corrected version (Good)
```text
# A single canonical source; the rest derive it, not copy it.
__version__ = importlib.metadata.version("mypkg")
```

### 🔎 Concrete detection
Extract all version strings from the repo (manifest, __version__, changelog,
tags) and fail if they do not match.
