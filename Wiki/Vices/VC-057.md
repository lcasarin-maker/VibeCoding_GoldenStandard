### Síntoma (Symptom)
> Documentos, ejecutores y estado declaran versiones distintas

### Causa (Cause)
Identidad distribuida inconsistente

### Solución (Solution)
Una versión canónica derivada, no copiada

### Acción Correctiva / Prevención
Behavioral/doctrinal vice — not statically falsifiable in a generic way. Documented in the Golden Standard catalogs as governance knowledge; no automated test can discriminate this without human semantic judgment. Sprint 3.4 triage: reclassified from AUDITED/test_behavioral_compliance to DOC_ONLY.

### ❌ Ejemplo del vicio (Bad)
```text
# Tres fuentes declaran versiones distintas.
pyproject.toml : version = "1.4.0"
__init__.py    : __version__ = "1.2.0"
CHANGELOG.md   : ## 1.5.0
```

### ✅ Versión corregida (Good)
```text
# Una sola fuente canonica; el resto la derivan, no la copian.
__version__ = importlib.metadata.version("mypkg")
```

### 🔎 Detección concreta
Extraer todas las cadenas de version del repo (manifiesto, __version__, changelog,
tags) y fallar si no coinciden.
