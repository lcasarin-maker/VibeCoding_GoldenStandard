### Symptom
> The model calls a method, parameter, or endpoint of a library that DOES exist, but that does not exist in the pinned version (deprecated, renamed, or invented from outdated knowledge).


### Cause
The model's knowledge is out of date with respect to the installed version; no one verifies the signature against the real API.


### Solution
Pin the version and verify each symbol against the real API (up-to-date docs, documentation MCP, or import + inspection); reject calls that do not resolve.


### Corrective Action / Prevention
Documented; enforcement downstream. Distinct from VC-061 (nonexistent package): here the package exists but the symbol does not. Mitigation: up-to-date docs via context7 (PR-110).


### ❌ Example of the vice (Bad)
```python
# pandas 2.x removed DataFrame.append; the model keeps using it.
df = df.append(row)            # AttributeError in the installed version
```

### ✅ Corrected version (Good)
```python
# Verify against the real API of the pinned version and use the current replacement.
df = pd.concat([df, pd.DataFrame([row])], ignore_index=True)
```

### 🔎 Concrete detection
Resolve each called symbol (method/attribute/kwarg/endpoint) against the installed version's API; a missing symbol = hallucinated or obsolete API. No self-contained detector (needs the library); see VC-061 and PR-013.
