### Síntoma (Symptom)
> Protocolo falla parcialmente sin reportar qué dependencia falta

### Causa (Cause)
Precondiciones no verificadas en startup; el sistema asume entorno completo

### Solución (Solution)
Validación activa de stack al inicio: runtime, CLI tools, hooks, archivos esenciales, permisos de escritura

### Acción Correctiva / Prevención
Checked by setup_validate.py which runs comprehensive pre-flight verification of Python, git hooks, write access, encoding, and the project registry.

### ❌ Ejemplo del vicio (Bad)
```python
# Se traga parte del stack: el test pasa aunque una etapa fallo en silencio.
def test_flow():
    try:
        a(); b(); c()
    except StepError:
        pass            # 'c' pudo no ejecutarse y nadie lo sabe
```

### ✅ Versión corregida (Good)
```python
def test_flow():
    assert a().ok
    assert b().ok
    assert c().ok       # cada etapa verificada, sin captura muda
```

### 🔎 Detección concreta
try/except que abarca multiples pasos y absorbe errores intermedios sin reafirmar
que cada paso se completo.

### 📚 Evidencia externa
