### Síntoma (Symptom)
> Un `ImportError` desencadena `pip install` sin revisión humana

### Causa (Cause)
El error handler delega decisiones de supply chain al entorno sin control

### Solución (Solution)
Prohibir `pip install` automático; reportar la dependencia faltante con guía de instalación manual y detener; el humano decide qué instala

### Acción Correctiva / Prevención
Disabled automatic subprocess pip installs in repair_failing_tests.py, forcing manual package guidance.

### ❌ Ejemplo del vicio (Bad)
```python
# Instalacion de dependencias en runtime, sin fijar ni verificar.
import os
os.system("pip install some-pkg")   # version arbitraria, codigo sin auditar
import some_pkg
```

### ✅ Versión corregida (Good)
```python
# Dependencia declarada y fijada en el manifiesto; cero instalacion en runtime.
# pyproject.toml: dependencies = ["some-pkg==1.4.2"]
import some_pkg
```

### 🔎 Detección concreta
Detectar invocaciones a pip/npm/installer dentro del codigo de la aplicacion
(os.system, subprocess) en vez del manifiesto de dependencias.
