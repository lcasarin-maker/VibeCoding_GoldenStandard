### Symptom
> The rules engine executes YAML or config strings as Python code (`eval`/`exec`)

### Cause
The rules source is treated as trusted code without distinguishing data from instructions

### Solution
Replace eval with a dispatch table of explicitly registered functions; validate names at load time; reject any unregistered check

### Corrective Action / Prevention
Replaced eval-based rules with a pre-registered SAFE_CHECKS dispatch table in rules_engine.py to prevent remote code execution.

### ❌ Example of the vice (Bad)
```python
# Dynamic evaluation of an expression from an external source: RCE.
rule = request.args["expr"]
if eval(rule):          # attacker-controlled input
    grant_access()
```

### ✅ Corrected version (Good)
```python
# Explicit, bounded parsing; never eval on external input.
import ast
rule = request.args["expr"]
value = ast.literal_eval(rule)   # literals only, no execution
if validate_rule(value):
    grant_access()
```

### 🔎 Concrete detection
AST: calls to eval()/exec()/compile() whose argument comes (by data flow)
from external input (request, file, network, argv).
Tools: bandit (Python), semgrep (multi-language).
