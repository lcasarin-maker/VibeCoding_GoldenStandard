### Síntoma (Symptom)
> Motor de reglas ejecuta strings de YAML o config como código Python (`eval`/`exec`)

### Causa (Cause)
La fuente de reglas se trata como código confiable sin distinguir datos de instrucciones

### Solución (Solution)
Reemplazar eval con dispatch table de funciones registradas explícitamente; validar nombres en tiempo de carga; rechazar cualquier check no registrado

### Acción Correctiva / Prevención
Replaced eval-based rules with a pre-registered SAFE_CHECKS dispatch table in rules_engine.py to prevent remote code execution.

### ❌ Ejemplo del vicio (Bad)
```python
# Evaluacion dinamica de una expresion de origen externo: RCE.
rule = request.args["expr"]
if eval(rule):          # entrada controlada por el atacante
    grant_access()
```

### ✅ Versión corregida (Good)
```python
# Parseo explicito y acotado; nunca eval sobre entrada externa.
import ast
rule = request.args["expr"]
value = ast.literal_eval(rule)   # solo literales, sin ejecucion
if validate_rule(value):
    grant_access()
```

### 🔎 Detección concreta
AST: llamadas a eval()/exec()/compile() cuyo argumento proviene (por flujo de datos)
de entrada externa (request, archivo, red, argv).
Tools: bandit (Python), semgrep (multi-lenguaje).
