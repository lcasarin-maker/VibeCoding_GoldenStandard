### Symptom
> Logic precedes the data model

### Cause
Absent causal structure

### Solution
Define entities and relations first

### Corrective Action / Prevention
Code accesses fields that are not declared in the schema or model, leading to
runtime AttributeError and silent data corruption when the model changes.


### ❌ Example of the vice (Bad)
```python
# Accessing a field that does not exist on the model.
user = User(name="Alice")
print(user.age)  # AttributeError: 'User' has no attribute 'age'
```

### ✅ Corrected version (Good)
```python
# Model declares all fields; access is safe.
from pydantic import BaseModel
class User(BaseModel):
    name: str
    age: int
user = User(name="Alice", age=30)
print(user.age)  # OK
```

### 🔎 Concrete detection
AST + type checker: access to attributes not declared in the class/schema.
Tools: mypy (attr-defined), pylint (E1101), Pydantic validation.
