### Symptom
> Cumulative additive mutations that scatter the system's functionality across multiple redundant files or unnecessary wrappers instead of consolidating logic.

### Cause
Generative convenience in creating new interfaces instead of restructuring the existing ones.

### Solution
Impose strict static validation that rejects the existence of unregistered wrappers or middlemen that duplicate more than 30% of the core functionality. Before adding a new function, verify the stdlib or an already-installed dependency solves it (ponytail review tag — stdlib).

### Corrective Action / Prevention
Top-level function names must be unique across modules: a structural comparison fails when the same definition name is duplicated, surfacing redundant slop.


### ❌ Example of the vice (Bad)
```python
# Instead of using the existing function, the agent creates a redundant wrapper
# that reimplements the same thing "for convenience". The logic scatters.
def get_user(id): ...           # already exists

def fetch_user(id):             # new wrapper: 90% duplicated
    u = get_user(id)
    return u

def retrieve_user(id):          # yet another, same AI-slop pattern
    return get_user(id)
```

### ✅ Corrected version (Good)
```python
# A single canonical function. If behavior must vary, parameterize.
def get_user(id, *, with_profile=False): ...
```

### 🔎 Concrete detection
AST: detect top-level functions whose body delegates >30% to an already
existing function without added logic (trivial wrappers) or synonymous names
(`get_`/`fetch_`/`retrieve_`) over the same symbol. Secondary: grep for hand-rolled
implementations of stdlib primitives (date parsing, base64, JSON merge, retry loops,
sorting, dedup) — flag with stdlib: tag and name the stdlib replacement.
