### Symptom
> Erroneous, hallucinated, or contradictory information enters the context and self-reinforces in following turns, degrading all subsequent responses.


### Cause
The context is not purged of unverified claims; noise accumulates and the model treats it as established truth.


### Solution
Verify before persisting; compact by removing the unconfirmed; re-anchor in sources of truth after each compaction.


### Corrective Action / Prevention
Documented; enforcement downstream. Recipe: a verification step before persisting a fact, and re-anchoring in sources of truth when compacting.


### ❌ Example of the vice (Bad)
```python
# The agent accepts a hallucinated datum and drags it along as truth.
ctx.append("the endpoint is /v2/pay")   # never verified; it was /v1/pay
# 5 turns later the agent keeps building on /v2/pay
```

### ✅ Corrected version (Good)
```python
# Verify before persisting; the unconfirmed does not enter the durable context.
if endpoint_exists("/v2/pay"):
    ctx.append_fact("endpoint=/v2/pay", source="openapi.json")
```

### 🔎 Concrete detection
Unverified claims that persist in the context and are cited as facts in later turns; the absence of a verification step before compacting. Related to VC-051 (saturation) and VC-001 (output as hypothesis).
