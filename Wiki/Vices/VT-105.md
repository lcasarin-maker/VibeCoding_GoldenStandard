### Symptom
> A pipeline with no verification that git hooks are installed and executable

### Cause
A missing hook passes CI validation because CI does not ask about hooks

### Solution
Check the presence and executability of each required hook as an explicit step of the suite

### Corrective Action / Prevention
Validated by tests/test_infrastructure.py:test_pre_commit_hook_exists_and_executable, ensuring hooks are physically present and active.

### ❌ Example of the vice (Bad)
```python
# There are critical git/CI hooks but no test verifies they exist/run.
# (no test) -> someone deletes .pre-commit-config and nobody notices
```

### ✅ Corrected version (Good)
```python
def test_precommit_hooks_present():
    cfg = Path(".pre-commit-config.yaml")
    assert cfg.exists()
    assert "validate_golden_standard_catalogs" in cfg.read_text()
```

### 🔎 Concrete detection
Absence of a test that asserts the existence and configuration of verification
hooks declared as part of the repo contract.
