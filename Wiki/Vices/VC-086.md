### Symptom
> Planning, execution, or debt accounting begins without a tracked backlog entry, so the work is effectively invisible to the planning gate.

### Cause
The agent treats unregistered work as admissible and starts reasoning before the ledger exists.

### Solution
Require a backlog entry before planning or execution; if it is not in the ledger, it does not get worked.

### Corrective Action / Prevention
Require a backlog entry before planning or execution; if it is not in the ledger, it does not get worked.


### тЭМ Example of the vice (Bad)


### тЬЕ Corrected version (Good)

### ЁЯФО Concrete detection
Flags planning or implementation that proceeds without a tracked backlog entry in the source ledger.
