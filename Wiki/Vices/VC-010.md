### Symptom
> A destructive action runs directly

### Cause
Lack of simulation

### Solution
Simulate before applying

### Corrective Action / Prevention
Static detector `vc036_destructive_without_dryrun` flags destructive operations that appear without a dry-run or simulation step in the same flow.

### ❌ Example of the vice (Bad)
```bash
# Mass deletion run directly, without simulating.
find . -name "*.tmp" -delete
```

### ✅ Corrected version (Good)
```bash
# Simulate first; apply only after reviewing the list.
find . -name "*.tmp" -print     # dry-run: what would be deleted
find . -name "*.tmp" -delete    # apply after confirming
```

### 🔎 Concrete detection
Destructive operations (rm/-delete/DROP/TRUNCATE/git reset --hard) without a prior
simulation step (--dry-run/-print/EXPLAIN) in the same flow.
