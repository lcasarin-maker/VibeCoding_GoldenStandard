### Symptom
> In long sessions the agent forgets or contradicts prior architectural decisions, reintroducing patterns it itself deprecated.


### Cause
Non-externalized decisions plus window saturation: the model does not re-anchor the active constraints as the context grows.


### Solution
Externalize decisions (ADR/DECISIONS.md), re-anchor after compacting, and validate every change against the decision record.


### Corrective Action / Prevention
Documented; enforcement downstream. Recipe: persistent ADR + post-compaction re-anchoring + coherence check against active decisions.


### ❌ Example of the vice (Bad)
```text
# Turn 10: 'we will use repositories, no SQL in the handlers'.
# Turn 80 (after compacting): the agent puts raw SQL in a handler.
```

### ✅ Corrected version (Good)
```text
# The decision lives in ADR-003 and is re-anchored after compacting; the change is validated against it.
# ADR-003: data access only via repos -> the handler with raw SQL is rejected.
```

### 🔎 Concrete detection
Changes that contradict recorded decisions (ADR/DECISIONS.md) without justification; the frequency grows with session length. Related to VC-051 and VC-034.

### 📚 External evidence
- **HumanLayer ACE** — decision drift in long sessions is a central motive for 'Frequent Intentional Compaction'.
