### Síntoma (Symptom)
> Nombre de componente no refleja alcance real tras evolución (ej. un componente denominado "auditor_v1" que ha sido expandido para cubrir funcionalidades de la versión v3 pero conserva su nombre heredado)

### Causa (Cause)
El nombre se fijó en un estado histórico y no se actualizó al expandir

### Solución (Solution)
Actualizar nombre y todos los tests que lo referencian al cambiar el alcance; o documentar la discrepancia explícitamente con centinela

### Acción Correctiva / Prevención
Behavioral/doctrinal vice — not statically falsifiable in a generic way. Documented in the Golden Standard catalogs as governance knowledge; no automated test can discriminate this without human semantic judgment. Sprint 3.4 triage: reclassified from AUDITED/test_behavioral_compliance to DOC_ONLY.

### ❌ Ejemplo del vicio (Bad)
```python
# El nombre dice v1 pero ya implementa reglas de la v3: enga;a a lectores y tests.
class AuditorV1:           # alcance real != nombre heredado
    ...
```

### ✅ Versión corregida (Good)
```python
class Auditor:             # el nombre refleja el alcance actual
    ...
```

### 🔎 Detección concreta
Identificadores con tokens de version embebidos (xxx_v1, FooV2) cuyo alcance ya no
corresponde a esa version.
