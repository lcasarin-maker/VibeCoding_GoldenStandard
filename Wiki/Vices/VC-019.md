### Symptom
> Loss of commits and regressions from parallel overwrites

### Cause
Divergent mental models from skipping prior filesystem synchronization

### Solution
Require a mandatory physical reconciliation step against the ledger before writing

### Corrective Action / Prevention
State must stay consistent across concurrent sessions: a parity check fails when the protocol fingerprint diverges between two sessions.


### ❌ Example of the vice (Bad)
```python
# Two sessions/agents write the same state without coordination.
# The last write wins and the other's work is lost (lost update).
state = load_state()
state["x"] = compute()
save_state(state)  # without checking version: overwrites concurrent changes
```

### ✅ Corrected version (Good)
```python
# Optimistic locking: the write fails if the version changed in between.
state = load_state()
base_version = state["version"]
state["x"] = compute()
save_state_if_unchanged(state, expected_version=base_version)
```

### 🔎 Concrete detection
Detect read->modify->write on shared state without a version token,
lock, or intermediate CAS (compare-and-swap).
