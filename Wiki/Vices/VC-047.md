### Symptom
> Component name does not reflect the real scope after evolution (e.g. a component named "auditor_v1" that has been expanded to cover v3 functionality but keeps its inherited name)

### Cause
The name was fixed in a historical state and was not updated when expanding

### Solution
Update the name and all tests that reference it when the scope changes; or document the discrepancy explicitly with a sentinel

### Corrective Action / Prevention
Static detector `vc047_frozen_nomenclature` flags identifiers that still carry embedded version tokens.

### ❌ Example of the vice (Bad)
```python
# The name says v1 but it already implements v3 rules: misleads readers and tests.
class AuditorV1:           # real scope != inherited name
    ...
```

### ✅ Corrected version (Good)
```python
class Auditor:             # the name reflects the current scope
    ...
```

### 🔎 Concrete detection
Identifiers with embedded version tokens (xxx_v1, FooV2) whose scope no longer
corresponds to that version.
