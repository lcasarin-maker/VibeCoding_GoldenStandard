### Symptom
> Returns the expected value

### Cause
Optimization against the example

### Solution
Test general properties

### Corrective Action / Prevention
A test must assert general properties, not a single hardcoded return value that the implementation can trivially satisfy.


### ❌ Example of the vice (Bad)
```python
# The function returns a constant to 'pass' the test; it implements nothing.
def total(cart):
    return 42          # hardcoded so the test goes green

def test_total():
    assert total(demo_cart) == 42
```

### ✅ Corrected version (Good)
```python
# The test fixes the real expected result; the function must compute it.
def total(cart):
    return sum(i.price for i in cart.items)

def test_total():
    assert total(Cart([Item(10), Item(32)])) == 42
```

### 🔎 Concrete detection
AST: function under test whose body is a single return of a constant literal, with a
test that asserts exactly that literal.
