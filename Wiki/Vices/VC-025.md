### Symptom
> Invalid data enters the system

### Cause
Permeable boundary

### Solution
Validate at boundaries

### Corrective Action / Prevention
Static detector `vc025_io_without_validation` flags external input that reaches a sink without boundary validation.

### ❌ Example of the vice (Bad)
```python
age = int(request.args["age"])                 # external input used without validation
db.execute(f"SELECT * FROM u LIMIT {request.args['n']}")  # unsanitized
```

### ✅ Corrected version (Good)
```python
age = validate_int(request.args.get("age"), lo=0, hi=120)   # validation at the boundary
db.execute("SELECT * FROM u LIMIT ?", (safe_int(request.args.get("n")),))
```

### 🔎 Concrete detection
Data from external sources (request/file/network/argv) that reaches a sink without going
through validation/schema at the boundary.
Tools: semgrep and CodeQL (taint mode) — the only ones with real dataflow.
