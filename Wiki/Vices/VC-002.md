### Symptom
> The temporary becomes permanent

### Cause
Lack of an expiration boundary

### Solution
Every prototype must have a date, owner, and exit

### Corrective Action / Prevention
Behavioral/doctrinal vice — not statically falsifiable in a generic way. Documented in the Golden Standard catalogs as governance knowledge; no automated test can discriminate this without human semantic judgment. Sprint 3.4 triage: reclassified from AUDITED/test_behavioral_compliance to DOC_ONLY.

### ❌ Example of the vice (Bad)
```python
# "Temporary" prototype with no date or owner: it stays forever.
def quick_hack():            # TODO: temporary, remove
    return cached_guess()
```

### ✅ Corrected version (Good)
```python
# Every prototype carries explicit, trackable expiration and owner.
def quick_hack():            # PROTOTYPE owner=luis expires=2026-07-01 ticket=BUG-12
    return cached_guess()
```

### 🔎 Concrete detection
Flag TODO/FIXME/HACK/'temporary'/'prototype' comments without a parseable owner +
expiration date + ticket.
