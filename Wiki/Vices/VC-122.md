### Symptom
> Automatic installation of arbitrary packages or dynamic imports that alter the environment without consent.

### Cause
Lack of preventive control in the pre-execution hooks.

### Solution
Physically block any command that attempts to perform package downloads or executions of URLs that are not explicitly declared and approved.

### Corrective Action / Prevention
Automatic pip install via subprocess in scripts/ blocked by a detector + active-path repo-guard.

### ❌ Example of the vice (Bad)
```bash
# Runs a remote script without pinning or verifying: the provider can inject code.
curl -s https://example.com/install.sh | bash
```

### ✅ Corrected version (Good)
```bash
# Download, pin by hash, and verify before executing.
curl -sSfL https://example.com/install.sh -o install.sh
echo "<sha256-conocido>  install.sh" | sha256sum -c -
bash install.sh
```

### 🔎 Concrete detection
Detect 'curl|bash' patterns, unpinned postinstall scripts, or execution of
downloaded binaries without hash/signature verification.
Tools (dep CVEs): Trivy (PI-004), OWASP DependencyCheck, Snyk.
