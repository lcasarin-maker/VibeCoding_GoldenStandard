### Symptom
> Runs without verifying

### Cause
Empty metric

### Solution
Coverage with oracles

### Corrective Action / Prevention
Coverage means executed-and-verified: lines run without an oracle assertion do not constitute coverage.


### ❌ Example of the vice (Bad)
```python
# Calls everything to raise coverage, but asserts nothing.
def test_pipeline():
    run_stage_a(); run_stage_b(); run_stage_c()   # 100% lines, 0% verification
```

### ✅ Corrected version (Good)
```python
def test_pipeline():
    out = run_stage_c(run_stage_b(run_stage_a(seed)))
    assert out.status == "done" and out.rows == 3
```

### 🔎 Concrete detection
Cross coverage with assert density: files with high coverage and tests without
active asserts = inflated coverage.

### 📚 External evidence
- **[[Evidence/arxiv2602_00409|arxiv:2602.00409]]** — an empirical study confirms that agents generate tests without an oracle (no assert, tautological, or over-mocked) in a measurable way.
