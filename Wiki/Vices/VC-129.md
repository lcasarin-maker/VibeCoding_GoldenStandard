### Síntoma (Symptom)
> El agente importa o declara un paquete que no existe en el registry (nombre inventado o casi-correcto), exponiendo a slopsquatting: un atacante puede registrar ese nombre con codigo malicioso.


### Causa (Cause)
El modelo genera nombres de paquetes plausibles pero inexistentes; nadie verifica su existencia ni legitimidad en el registry oficial.


### Solución (Solution)
Verificar que cada dependencia existe en el registry oficial, evaluar su popularidad/legitimidad y fijar version; rechazar nombres que no resuelven.


### Acción Correctiva / Prevención
Documentado; enforcement downstream. Hueco confirmado: ningun SCA tradicional (Trivy, Snyk, OWASP DependencyCheck) detecta paquetes inexistentes, solo CVEs de paquetes reales.


### ❌ Ejemplo del vicio (Bad)
```python
# El modelo 'alucina' un paquete que no existe en PyPI.
import nicethings              # no existe -> alguien puede registrarlo con malware
# pip install reqests          # typo de 'requests' (typosquatting)
```

### ✅ Versión corregida (Good)
```python
# Verificar existencia y legitimidad en el registry antes de declarar.
#   pip index versions requests   # resuelve -> real, popular
#   nombre que no resuelve en PyPI = rechazar, no instalar
```

### 🔎 Detección concreta
Paquetes importados/declarados que no resuelven en el registry oficial del lenguaje, o cuyo nombre es un typo cercano a un paquete popular. Distinto de VC-082 (usado pero no declarado) y VC-116 (auto-instalacion). No cubierto por SCA tradicional.
