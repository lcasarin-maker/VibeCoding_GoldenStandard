### Síntoma (Symptom)
> Se ordena reemplazar X por Y. El agente crea: herencia de X en Y, shim que reenvía a Y, fallback `Y.exists() or X.exists()`, centinelas duales en tests, imports cruzados "por retrocompatibilidad". X nunca muere.

### Causa (Cause)
El agente interpreta "reemplazar" como "añadir" para evitar regresar a corregir fallos. El instinto de "no romper nada" genera código muerto disfrazado de seguridad.

### Solución (Solution)
**Reemplazar = Eliminar + Crear. Una sola fuente de verdad.** Cuando se ordena reemplazar: (1) el archivo viejo se ELIMINA del VCS, (2) el nuevo no importa del viejo, (3) los tests apuntan solo al nuevo, (4) no existen rutas alternativas ni checks duales. Cualquier "por compatibilidad" en código activo es señal de reemplazo incompleto.

### Acción Correctiva / Prevención
Prevented by D1 _audit_d1_zombie_compat scanning active scripts for zombie compatibility shim patterns.

### ❌ Ejemplo del vicio (Bad)
```python
# Shim de compatibilidad muerto: nadie lo usa pero sigue 'por si acaso'.
def legacy_api(*a, **k):
    return new_api(*a, **k)   # zombie: solo reenvia, jamas se llama
```

### ✅ Versión corregida (Good)
```python
# Se elimina el shim y se migran los pocos llamadores reales (si los hay).
# grep confirma 0 referencias activas -> borrar, no conservar.
def new_api(...):
    ...
```

### 🔎 Detección concreta
grep de referencias: funciones/modulos marcados 'legacy'/'compat'/'deprecated' con
0 llamadores activos en el arbol = candidatos zombie a eliminar.
