### Síntoma (Symptom)
> Mutaciones aditivas acumulativas que dispersan la funcionalidad del sistema en múltiples archivos redundantes o wrappers innecesarios en vez de consolidar lógica.

### Causa (Cause)
Comodidad generativa al crear nuevas interfaces en lugar de reestructurar las existentes.

### Solución (Solution)
Imponer una validación estática estricta que rechace la existencia de wrappers o intermediarios no registrados que dupliquen más del 30% de la funcionalidad core.

### Acción Correctiva / Prevención
Nombres de función top-level duplicados entre módulos detectados por comparación AST.

### ❌ Ejemplo del vicio (Bad)
```python
# En vez de usar la función existente, el agente crea un wrapper redundante
# que reimplementa lo mismo "por comodidad". La lógica se dispersa.
def get_user(id): ...           # ya existe

def fetch_user(id):             # wrapper nuevo: 90% duplicado
    u = get_user(id)
    return u

def retrieve_user(id):          # otro más, mismo patrón AI slop
    return get_user(id)
```

### ✅ Versión corregida (Good)
```python
# Una sola función canónica. Si hace falta variar comportamiento, parametrizar.
def get_user(id, *, with_profile=False): ...
```

### 🔎 Detección concreta
AST: detectar funciones top-level cuyo cuerpo delega >30% en otra función ya
existente sin lógica añadida (wrappers triviales) o nombres sinónimos
(`get_`/`fetch_`/`retrieve_`) sobre el mismo símbolo.

### 📚 Evidencia externa
- **GitClear 2025** — el código duplicado/clonado aumentó ~8x con asistencia de IA, señal directa de este antipatrón a escala.
