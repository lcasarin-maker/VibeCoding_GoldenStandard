### Symptom
> Tests that always pass (green) regardless of what logic changes or breakages are inserted into production code.

### Cause
Happy-path assertion theater and self-satisfied mocks.

### Solution
Require that at least one test suite empirically demonstrate its ability to fail (falsifiability criterion) against a controlled code mutation or chaos injection.

### Corrective Action / Prevention
A suite must empirically demonstrate it can fail against a controlled mutation; tests that stay green under any change are not falsifiable.


### ❌ Example of the vice (Bad)
```text
# No code mutation makes the suite fail.
```

### ✅ Corrected version (Good)
```text
# mutation testing: the suite must kill mutants
```

### 🔎 Concrete detection
Suite without mutation validation: it is not proven to detect injected defects.
Tools: mutmut / cosmic-ray (Python), Stryker (JS/TS), PITest (Java).

- **arxiv:2602.00409** — an empirical study confirms that agents generate tests without an oracle (no assert, tautological, over-mocked) in a measurable way.
