### Symptom
> Automated git synchronization commands take indefinitely long, lock the disk (Google Drive virtual mounts), or try to track thousands of temporary/hidden files created by external tools.

### Cause
Use of `git add -A` or `git add .` without filtering or previously validating the working-tree state against excluded directories or virtualized cloud storage.

### Solution
Prefer `git add -u` to record modifications to files under version control and require surgical explicit additions for new files, ensuring that `.gitignore` is loaded before any Git subprocess invocation.

### Corrective Action / Prevention
Staging must be deliberate: indiscriminate staging of all untracked files is rejected, requiring explicit human intent about what enters a commit.


### ❌ Example of the vice (Bad)
```bash
# Indiscriminate staging: drags in secrets, artifacts, and untracked junk.
git add .
git commit -m "wip"
```

### ✅ Corrected version (Good)
```bash
# Explicit staging of what was changed on purpose; .gitignore covers the rest.
git add src/feature.py tests/test_feature.py
git commit -m "feat: add feature with tests"
```

### 🔎 Concrete detection
Pre-commit hook: alert on 'git add .'/'git add -A' or staging of files that
match secret/artifact patterns not explicitly listed.
