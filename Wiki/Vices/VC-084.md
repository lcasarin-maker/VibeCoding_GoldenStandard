### Symptom
> The same truth is corrected in multiple tracked files, or a rendered output is hand-patched instead of regenerated from the canonical source.

### Cause
The agent treats derived artifacts as edit targets instead of outputs owned by the source or generator.

### Solution
Edit only the canonical source or generator, regenerate all derived artifacts, and verify sink parity before closing.

### Corrective Action / Prevention
Edit the canonical source or its generator first: hand-patching a derived artifact to mask drift is rejected; derived files must be regenerated from source.


### ❌ Example of the vice (Bad)
```yaml
# Fixing the rendered wiki directly while leaving the source catalog stale.
title: "Canonical source first (no sink patching)"
status: AUDITED
```

### ✅ Corrected version (Good)
```yaml
# Fix the source catalog, then regenerate the derived artifacts.
title: "Canonical source first (no sink patching)"
status: AUDITED
```

### 🔎 Concrete detection
Flags direct edits to generated wiki/report files when the source catalog or generator was not the thing that changed.
