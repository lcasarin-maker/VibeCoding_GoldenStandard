### Symptom
> The same truth is corrected in multiple tracked files, or a rendered output is hand-patched instead of regenerated from the canonical source.

### Cause
The agent treats derived artifacts as edit targets instead of outputs owned by the source or generator.

### Solution
Edit only the canonical source or generator, regenerate all derived artifacts, and verify sink parity before closing. For bug fixes, grep every caller of the touched function and fix the shared function once — one guard at the source is smaller than one per caller, and patching only the reported call-site leaves sibling callers broken.

### Corrective Action / Prevention
Edit the canonical source or its generator first: hand-patching a derived artifact to mask drift is rejected; derived files must be regenerated from source.


### ❌ Example of the vice (Bad)
```yaml
# Fixing the rendered wiki directly while leaving the source catalog stale.

```

### ✅ Corrected version (Good)
```yaml
# Fix the source catalog, then regenerate the derived artifacts.
title: "Canonical source first (no sink patching)"
status: DOC_ONLY
```

### 🔎 Concrete detection
Flags direct edits to generated wiki/report files when the source catalog or generator was not the thing that changed. Also flags bug fixes that patch only the call-site reported in the ticket while grep shows other callers of the same function remain unpatched.
