### Symptom
> Undeclared artifacts survive

### Cause
Non-binding inventory

### Solution
Every dead file must be authorized, classified, or deleted

### Corrective Action / Prevention
Static integrity analysis enumerates every code file (excluding non-code by
extension) and fails with "unregistered file (dead)" when a file is neither
declared in the authorized inventory (whitelist) nor a legitimate test. The
signature is presence-on-disk without any authorizing reference.


### ❌ Example of the vice (Bad)
```text
# old_server.py, server_v2_backup.py coexist without being declared or used.
```

### ✅ Corrected version (Good)
```text
# Each artifact is authorized, classified, or deleted (explicit inventory).
```

### 🔎 Concrete detection
Files not referenced by imports/build nor declared in an inventory = dead files.

- **arxiv:2603.28592** — recurring anti-pattern of AI-generated code, present in 80-100% of the analyzed projects.
