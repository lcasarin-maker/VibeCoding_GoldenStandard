### Síntoma (Symptom)
> Acción destructiva se ejecuta directa

### Causa (Cause)
Falta de simulación

### Solución (Solution)
Simular antes de aplicar

### Acción Correctiva / Prevención
Behavioral/doctrinal vice — not statically falsifiable in a generic way. Documented in the Golden Standard catalogs as governance knowledge; no automated test can discriminate this without human semantic judgment. Sprint 3.4 triage: reclassified from AUDITED/test_behavioral_compliance to DOC_ONLY.

### ❌ Ejemplo del vicio (Bad)
```bash
# Borrado masivo ejecutado directo, sin simular.
find . -name "*.tmp" -delete
```

### ✅ Versión corregida (Good)
```bash
# Simular primero; aplicar solo tras revisar la lista.
find . -name "*.tmp" -print     # dry-run: que se borraria
find . -name "*.tmp" -delete    # aplicar tras confirmar
```

### 🔎 Detección concreta
Operaciones destructivas (rm/-delete/DROP/TRUNCATE/git reset --hard) sin un paso
previo de simulacion (--dry-run/-print/EXPLAIN) en el mismo flujo.
