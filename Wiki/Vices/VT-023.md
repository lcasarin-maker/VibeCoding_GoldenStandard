### Symptom
> Simulates success

### Cause
Reality substituted

### Solution
Calibrate with integration

### Corrective Action / Prevention
A mock that always simulates success must be calibrated against a real integration, or it only tests itself.


### ❌ Example of the vice (Bad)
```python
api = Mock(); api.charge.return_value = True   # never fails
```

### ✅ Corrected version (Good)
```python
# contract test against real behavior, including its failures
```

### 🔎 Concrete detection
Mock that always returns success, without calibrating against real integration.
