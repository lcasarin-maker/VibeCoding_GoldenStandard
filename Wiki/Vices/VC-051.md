### Symptom
> A crash or kill during `write_text()` leaves the state file (JSON, YAML) corrupt or empty

### Cause
Write atomicity was not modeled as a requirement of the persistence subsystem

### Solution
Write to a temporary file in the same directory and rename atomically (`tmp → final`); clean up the temporary if an exception occurs before the rename

### Corrective Action / Prevention
Writes to critical state must be atomic: state is persisted via a temp-file-plus-atomic-replace path so a crash cannot leave a half-written, corrupted file.


### ❌ Example of the vice (Bad)
```python
# Non-atomic write: if the process dies midway, the critical file corrupts.
with open("state.json", "w") as f:
    json.dump(big_state, f)   # truncated and without atomic replacement
```

### ✅ Corrected version (Good)
```python
# Write to a temp and replace atomically (os.replace is atomic on the same FS).
import os, tempfile, json
fd, tmp = tempfile.mkstemp(dir=".")
with os.fdopen(fd, "w") as f:
    json.dump(big_state, f)
os.replace(tmp, "state.json")
```

### 🔎 Concrete detection
Detect open(path,'w')/direct truncation on critical state files without the
write-temp + atomic os.replace/rename pattern.
