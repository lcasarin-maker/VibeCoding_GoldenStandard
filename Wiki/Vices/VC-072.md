### Symptom
> The next agent re-derives the context from scratch, spends tokens, or strays from the task.

### Cause
Changes committed without an explicit, fresh handoff for the next agent.

### Solution
A living HANDOFF.md with a fixed schema, mandatory and mechanically verified on every commit (agent-agnostic).

### Corrective Action / Prevention
Handoff continuity is mandatory: a commit with substantive changes is blocked when the handoff record was not updated or is missing its required sections; an explicit token allows a conscious skip.


### ❌ Example of the vice (Bad)
```markdown
# commit with new code and HANDOFF.md untouched -> the next agent loses the thread
git commit -m "fix: parser"
```

### ✅ Corrected version (Good)
```markdown
# HANDOFF.md updated and staged in the same commit, with the fixed schema
## ESTADO   parser fixed, suite green
## SIGUIENTE 1. migrate the lexer in scripts/lexer.py
## VERIFICAR pytest -q
```

### 🔎 Concrete detection
Substantive changes staged without a fresh HANDOFF.md in the changeset, or HANDOFF.md missing
the mandatory sections. Mechanically verified by check_handoff_freshness in commit-msg.
