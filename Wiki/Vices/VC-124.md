# VC-124: Deprecación Precipitada sin Análisis (Hasty Deprecation)

---

### Síntoma (Symptom)
> Un agente mueve un archivo a deprecated/ sin leerlo, sin buscar referencias activas y sin verificar si su funcionalidad sigue siendo útil o está cubierta en otro lugar.

### Causa (Cause)
El agente trata deprecated/ como papelera de reciclaje en lugar de cuarentena auditada. El impulso de "resolver el error rápido" supera el análisis requerido.

### Solución (Solution)
Proceso obligatorio antes de deprecar — (1) leer el archivo completo, (2) buscar referencias activas con grep, (3) verificar duplicidad funcional, (4) documentar decisión en DEPRECATION_LOG, (5) ENTONCES ejecutar git mv.

### Acción Correctiva / Prevención
pre_edit_guard bloquea escrituras en deprecated/ sin entrada previa en DEPRECATION_LOG.md. pre-commit hook valida que archivos staged en deprecated/ estén justificados. D1 del auditor 12D verifica cobertura del log.

### ❌ Ejemplo del vicio (Bad)
```bash
# Mueve a deprecated/ sin leer, sin buscar referencias, sin justificar.
git mv module.py deprecated/module.py
```

### ✅ Versión corregida (Good)
```bash
# Proceso auditado antes de deprecar.
grep -rn "import module" src/        # 1) referencias activas?
# 2) leer el archivo  3) verificar duplicidad  4) registrar la decision:
echo "module.py: sin refs, cubierto por core.py - 2026-06-05" >> DEPRECATION_LOG.md
git mv module.py deprecated/module.py
```

### 🔎 Detección concreta
Pre-edit guard: rechazar escrituras/moves hacia deprecated/ sin una entrada previa
correspondiente en DEPRECATION_LOG.md.
