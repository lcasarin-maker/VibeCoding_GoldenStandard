### Symptom
> The model's output (SQL, HTML, command, path, or code) is executed, rendered, or used in a sink without sanitizing, assuming it is safe.


### Cause
The LLM's output is treated as trusted; there is no boundary between 'generated text' and 'executed action'.


### Solution
Treat the LLM's output as UNtrusted: parameterize (SQL), escape (HTML), allowlist (commands/paths), and validate against a schema before executing.


### Corrective Action / Prevention
Static detector `vc071_blind_trust_in_llm_output` flags model output that reaches a dangerous sink without sanitization.

### ❌ Example of the vice (Bad)
```python
sql = model.generate("query for user " + uid)   # LLM output
cursor.execute(sql)                              # executed without sanitizing -> injection
```

### ✅ Corrected version (Good)
```python
# Never execute the raw output: derive validated parameters and parameterize.
filters = validate_schema(model.generate_json(...))
cursor.execute("SELECT * FROM t WHERE owner = ?", (filters['uid'],))
```

### 🔎 Concrete detection
Model output that flows to a dangerous sink (cursor.execute with a string, os.system, eval/exec, innerHTML) without sanitization. Flow-dependent -> delegates to semgrep/CodeQL (taint). See PR-020, PR-034, PR-041.
