### Symptom
> Idealized environment

### Cause
Ignored variability

### Solution
Pessimistic configuration validation

### Corrective Action / Prevention
Hardcoded configuration values that assume a perfect development environment
will break in production where the environment differs.


### ❌ Example of the vice (Bad)
```python
# Hardcoded development config that will fail in production.
DEBUG = True
DATABASE_URL = "sqlite:///dev.db"
API_KEY = "test-key-123"
```

### ✅ Corrected version (Good)
```python
# Configuration is read from environment with sensible defaults.
import os
DEBUG = os.environ.get("DEBUG", "False").lower() == "true"
DATABASE_URL = os.environ.get("DATABASE_URL", "sqlite:///dev.db")
API_KEY = os.environ.get("API_KEY")
assert API_KEY, "API_KEY must be set in production"
```

### 🔎 Concrete detection
Regex: hardcoded config values (DEBUG=True, DATABASE_URL=..., API_KEY=...) in
source files instead of environment-variable reads. Also: .env files committed to git.
