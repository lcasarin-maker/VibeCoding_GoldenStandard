### Symptom
> The expected value ends up copied inside the code or the fixture and stops being an independent observation

### Cause
The oracle is embedded in the implementation or in a shared fixture

### Solution
Keep the expected value out of the production path and validate against an external specification

### Corrective Action / Prevention
Audited as oracle-contamination in test fixtures or implementation glue; separate from stale-spec mismatches.

### ❌ Example of the vice (Bad)
```python
EXPECTED = compute(x)
assert compute(x) == EXPECTED   # the expected value comes from the SUT itself
```

### ✅ Corrected version (Good)
```python
assert compute(x) == spec_value(x)   # expected value from an external specification
```

### 🔎 Concrete detection
The expected value is derived from the SUT itself instead of an independent source.
