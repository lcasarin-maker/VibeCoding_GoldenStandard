### Symptom
> The agent imports or declares a package that does not exist in the registry (invented or near-correct name), exposing it to slopsquatting: an attacker can register that name with malicious code.


### Cause
The model generates plausible but nonexistent package names; no one verifies their existence or legitimacy in the official registry.


### Solution
Verify that each dependency exists in the official registry, assess its popularity/legitimacy, and pin the version; reject names that do not resolve.


### Corrective Action / Prevention
Declared dependencies must exist in the public registry: a lookup fails when a requirement resolves to not-found, catching hallucinated (slopsquatted) packages.


### ❌ Example of the vice (Bad)
```python
# The model 'hallucinates' a package that does not exist on PyPI.
import nicethings              # does not exist -> someone can register it with malware
# pip install reqests          # typo of 'requests' (typosquatting)
```

### ✅ Corrected version (Good)
```python
# Verify existence and legitimacy in the registry before declaring.
#   pip index versions requests   # resolves -> real, popular
#   a name that does not resolve on PyPI = reject, do not install
```

### 🔎 Concrete detection
Packages imported/declared that do not resolve in the language's official registry, or whose name is a near typo of a popular package. Distinct from VC-030 (used but not declared) and PR-035 (auto-installation). Not covered by traditional SCA.
