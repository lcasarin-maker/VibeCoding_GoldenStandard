### Síntoma (Symptom)
> Simulación temporal se vuelve dependencia

### Causa (Cause)
Sustituto confundido con componente

### Solución (Solution)
Retirar stubs antes de validar integración

### Acción Correctiva / Prevención
Behavioral/doctrinal vice — not statically falsifiable in a generic way. Documented in the Golden Standard catalogs as governance knowledge; no automated test can discriminate this without human semantic judgment. Sprint 3.4 triage: reclassified from AUDITED/test_behavioral_compliance to DOC_ONLY.

### ❌ Ejemplo del vicio (Bad)
```python
# Un stub temporal queda cableado como dependencia real de produccion.
class PaymentGateway:
    def charge(self, amount):
        return True        # stub: "luego lo conectamos"
```

### ✅ Versión corregida (Good)
```python
# El stub vive solo en tests; produccion usa la implementacion real.
class PaymentGateway:
    def charge(self, amount):
        return self._provider.charge(amount)
```

### 🔎 Detección concreta
Clases/funciones con cuerpo trivial (return True/None/pass) importadas desde modulos
de produccion (no de tests/fixtures).
