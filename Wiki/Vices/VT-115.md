# VT-115: Falso Positivo de Drift por Fin de Línea (CRLF/LF Hash Mismatch)

### Síntoma (Symptom)
> La auditoría de drift (D12) falla falsamente al comparar archivos idénticos en disco (Windows CRLF) contra repositorios Git o satélites (LF).

### Causa (Cause)
Comparación binaria directa de hashes SHA256 sin normalización previa de caracteres de fin de línea, ignorando las políticas de traducción automática de Git (`core.autocrlf`).

### Solución (Solution)
Normalizar obligatoriamente todo byte stream binario decodificándolo y convirtiendo la secuencia de fin de línea `\r\n` a `\n` antes de calcular el hash de verificación de integridad o drift.

### Acción Correctiva / Prevención
Checked by setup_validate.py which runs comprehensive pre-flight verification of Python, git hooks, write access, encoding, and the project registry.
