### Symptom
> The drift audit (D12) fails falsely when comparing files that are identical on disk (Windows CRLF) against Git repositories or satellites (LF).

### Cause
Direct binary comparison of SHA256 hashes without prior normalization of line-ending characters, ignoring Git's automatic translation policies (`core.autocrlf`).

### Solution
Mandatorily normalize every binary byte stream by decoding it and converting the `\r\n` line-ending sequence to `\n` before computing the integrity- or drift-verification hash.

### Corrective Action / Prevention
Normalize line endings to a single form before hashing so files identical on disk do not raise a false drift positive across platforms.


### ❌ Example of the vice (Bad)
```python
# False 'drift' positive: compares raw bytes and CRLF/LF triggers a difference.
def test_no_drift():
    assert open("a.txt","rb").read() == open("b.txt","rb").read()
```

### ✅ Corrected version (Good)
```python
# Normalize line endings before comparing logical content.
def test_no_drift():
    norm = lambda p: open(p, encoding="utf-8").read().replace("\r\n", "\n")
    assert norm("a.txt") == norm("b.txt")
```

### 🔎 Concrete detection
Byte-by-byte file comparisons that do not normalize EOL (CRLF vs LF) and produce
false drift in mixed Windows/Unix environments.
