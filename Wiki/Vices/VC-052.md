### Symptom
> X is ordered to be replaced by Y. The agent creates: inheritance of X in Y, a shim that forwards to Y, a fallback `Y.exists() or X.exists()`, dual sentinels in tests, cross imports "for legacy compatibility". X never dies.

### Cause
The agent interprets "replace" as "add" to avoid going back to fix failures. The "do not break anything" instinct generates dead code disguised as safety.

### Solution
**Replace = Delete + Create. A single source of truth.** When a replacement is ordered: (1) the old file is DELETED from the VCS, (2) the new one does not import from the old one, (3) tests point only to the new one, (4) no alternative paths or dual checks exist. Any "for compatibility" in active code is a sign of an incomplete replacement.

### Corrective Action / Prevention
A replacement is a single source of truth: the superseded unit is deleted and the new one carries no inheritance, shim, fallback, or dual test sentinel back to it; static analysis flags compatibility forwarders left in active code.


### ❌ Example of the vice (Bad)
```python
# Dead compat shim: no one uses it but it stays 'just in case'.
def legacy_api(*a, **k):
    return new_api(*a, **k)   # zombie: only forwards, never called
```

### ✅ Corrected version (Good)
```python
# The shim is removed and the few real callers (if any) are migrated.
# grep confirms 0 active references -> delete, do not keep.
def new_api(...):
    ...
```

### 🔎 Concrete detection
grep of references: functions/modules marked 'legacy'/'compat'/'deprecated' with
0 active callers in the tree = dead candidates to remove.
