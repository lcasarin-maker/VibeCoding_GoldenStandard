### Symptom
> An expected failure becomes permanent

### Cause
Temporary exception turned into a contract

### Solution
Forbid an expected-failure from counting as green

### Corrective Action / Prevention
An expected failure must not count as green; normalizing it makes the failure permanent.


### ❌ Example of the vice (Bad)
```python
# The failure is 'normalized' as expected so the suite stays green.
EXPECTED_FAILURES = {"test_refund", "test_tax"}   # list that only grows
```

### ✅ Corrected version (Good)
```python
# Each failure is fixed or tracked individually with cause and date; no global list.
@pytest.mark.xfail(reason="BUG-200", strict=True)
def test_refund(): ...
```

### 🔎 Concrete detection
Detect global 'expected failure' lists/allowlists that grow monotonically
without per-entry expiration.
