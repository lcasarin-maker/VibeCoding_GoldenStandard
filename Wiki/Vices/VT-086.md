### Síntoma (Symptom)
> Fallo esperado se vuelve permanente

### Causa (Cause)
Excepción temporal convertida en contrato

### Solución (Solution)
Prohibir expected failure como verde

### Acción Correctiva / Prevención
Checked by D9 Test Purity which rejects permanent xfail or skip markers unless annotated with removal criteria or reasons.

### ❌ Ejemplo del vicio (Bad)
```python
# El fallo se 'normaliza' como esperado para que el suite quede verde.
EXPECTED_FAILURES = {"test_refund", "test_tax"}   # lista que solo crece
```

### ✅ Versión corregida (Good)
```python
# Cada fallo se arregla o se trackea individualmente con causa y fecha; sin lista global.
@pytest.mark.xfail(reason="BUG-200", strict=True)
def test_refund(): ...
```

### 🔎 Detección concreta
Detectar listas/allowlists globales de 'fallos esperados' que crecen monotonamente
sin expiracion por entrada.
