### Síntoma (Symptom)
> Se declara éxito sin logs

### Causa (Cause)
Confusión entre afirmación y verificación

### Solución (Solution)
Prohibir veredictos sin evidencia observable

### Acción Correctiva / Prevención
Anti-triunfalismo: el test falla si no hay evidencia JSON en .protocol/evidence/, forzando prueba empírica.

### ❌ Ejemplo del vicio (Bad)
```python
# Se declara exito sin ninguna evidencia observable.
def deploy():
    do_release()
    print("Deploy OK")  # afirmacion, no verificacion
```

### ✅ Versión corregida (Good)
```python
# El veredicto exige evidencia: el flujo falla si no hay prueba.
def deploy():
    result = do_release()
    assert result.health_check().ok, "deploy sin health-check verde"
    write_evidence(".protocol/evidence/deploy.json", result.metrics())
```

### 🔎 Detección concreta
Marcar como vicio toda ruta que emita 'OK'/'exito'/'done' sin un assert sobre estado
observable ni artefacto de evidencia generado en la misma ejecucion.
