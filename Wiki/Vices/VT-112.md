### Symptom
> Importing and integrating packages and modules at coding time without explicitly registering their existence in the dependency manifest.

### Cause
Lack of static import-consistency checking in the pre-commit hooks.

### Solution
Design static gates (`deptry` or AST regex in Python) that compare 100% of the top-level imports against the project's dependency base.

### Corrective Action / Prevention
A static gate must compare every top-level import against the dependency manifest so an unregistered dependency cannot drift in.


### ❌ Example of the vice (Bad)
```text
# requirements lists libs no longer used and is missing others.
```

### ✅ Corrected version (Good)
```text
# deptry / AST gate: imports used == declared
```

### 🔎 Concrete detection
Drift between declared and actually used dependencies, without a gate.
