### Symptom
> The same model (or the same instance/context) generates the code and then 'reviews' it, approving its own biases and errors.


### Cause
Lack of an independent oracle: the reviewer shares the generator's prior and context, so it repeats the same blind spots.


### Solution
Independent reviewer: another model or version, or a static/human check that does not share the generation context.


### Corrective Action / Prevention
Documented; enforcement downstream. Recipe: separate generator and reviewer (distinct model/version/context) or add an independent static gate.


### ❌ Example of the vice (Bad)
```python
code = model.generate(spec)
verdict = model.review(code)        # same model and context: approves its biases
```

### ✅ Corrected version (Good)
```python
code = model_a.generate(spec)
verdict = model_b.review(code)      # independent oracle
assert static_checks(code).ok       # + static gate that does not share the prior
```

### 🔎 Concrete detection
Same model/instance/context used to generate and to review/approve the change, without an external oracle. Extends VT-081 (author tests their own implementation) to the agent case.

### 📚 External evidence
- **arxiv:2603.28592** — self-validation without an independent oracle lets systematic defects from the generator slip through.
