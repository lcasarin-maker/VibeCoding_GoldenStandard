### Síntoma (Symptom)
> El mismo modelo (o la misma instancia/contexto) genera el codigo y luego lo 'revisa', aprobando sus propios sesgos y errores.


### Causa (Cause)
Falta de oraculo independiente: el revisor comparte el prior y el contexto del generador, asi que repite sus mismos puntos ciegos.


### Solución (Solution)
Revisor independiente: otro modelo o version, o un check estatico/humano que no comparta el contexto de generacion.


### Acción Correctiva / Prevención
Documentado; enforcement downstream. Receta: separar generador y revisor (modelo/version/contexto distintos) o anadir una compuerta estatica independiente.


### ❌ Ejemplo del vicio (Bad)
```python
code = model.generate(spec)
verdict = model.review(code)        # mismo modelo y contexto: aprueba sus sesgos
```

### ✅ Versión corregida (Good)
```python
code = model_a.generate(spec)
verdict = model_b.review(code)      # oraculo independiente
assert static_checks(code).ok       # + compuerta estatica que no comparte prior
```

### 🔎 Detección concreta
Mismo modelo/instancia/contexto usado para generar y para revisar/aprobar el cambio, sin oraculo externo. Extiende VT-081 (autor prueba su implementacion) al caso agente.
