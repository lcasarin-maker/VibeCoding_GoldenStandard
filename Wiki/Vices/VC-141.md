### Symptom
> After a commit, the working tree is left with someone else's unintegrated changes; the next agent inherits silent debt and does not know whether it is intentional or an oversight.

### Cause
The agent makes a selective commit (git add of only its own files) and works around the previous pending changes instead of reviewing, fixing, and integrating them.

### Solution
Before closing the commit, the working tree must be clean: every pending change is reviewed (diff), fixed if it introduces debt, and integrated into the commit; or gitignored with an explicit decision. Mechanically verified.

### Corrective Action / Prevention
Physical verification: scripts/check_clean_worktree.py invoked in the git pre-commit hook
(agnostic: runs with any `git commit`). Blocks if after staging something is left undecided
in the working tree (modified non-staged or untracked non-ignored). The root cause
of auto-generated artifact churn is neutralized with idempotent writes and auto-staging
of a canonical whitelist. Conscious escape: env CERBERUS_ALLOW_PARTIAL=1.


### ❌ Example of the vice (Bad)
```bash
# another agent's previous uncommitted changes -> ignored and only one's own is committed
git add scripts/mine.py && git commit -m "feat: mine"   # leaves REGISTRY.json, other.py dangling
```

### ✅ Corrected version (Good)
```bash
# ALL pending changes are reviewed, fixed, and integrated (or gitignored)
git diff other.py            # review
git add scripts/mine.py other.py REGISTRY.json
git commit -m "feat: mine + integrate pending"   # clean working tree
```

### 🔎 Concrete detection
After the pre-commit, `git status --porcelain` reports non-staged or untracked non-ignored
files that did not enter the commit. Verified by check_clean_worktree in pre-commit.
