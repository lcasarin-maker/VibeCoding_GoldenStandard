### Síntoma (Symptom)
> Un crash o kill durante `write_text()` deja el archivo de estado (JSON, YAML) corrupto o vacío

### Causa (Cause)
La atomicidad de escritura no se modeló como requisito del subsistema de persistencia

### Solución (Solution)
Escribir a archivo temporal en el mismo directorio y renombrar atómicamente (`tmp → final`); limpiar el temporal si ocurre una excepción antes del rename

### Acción Correctiva / Prevención
Atomic state writes via core_utils.write_json_atomic (tempfile + os.replace + fsync), wired into update_lock_state and ProtocolSyncManager._save_state.

### ❌ Ejemplo del vicio (Bad)
```python
# Escritura no atomica: si el proceso muere a mitad, el archivo critico se corrompe.
with open("state.json", "w") as f:
    json.dump(big_state, f)   # truncado y sin reemplazo atomico
```

### ✅ Versión corregida (Good)
```python
# Escribe a temporal y reemplaza atomicamente (os.replace es atomico en el mismo FS).
import os, tempfile, json
fd, tmp = tempfile.mkstemp(dir=".")
with os.fdopen(fd, "w") as f:
    json.dump(big_state, f)
os.replace(tmp, "state.json")
```

### 🔎 Detección concreta
Detectar open(path,'w')/truncado directo sobre archivos de estado critico sin patron
write-temp + os.replace/rename atomico.
