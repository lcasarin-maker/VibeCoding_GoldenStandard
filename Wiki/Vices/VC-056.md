### Symptom
> An agent moves a file to deprecated/ without reading it, without searching for active references, and without verifying whether its functionality is still useful or covered elsewhere.

### Cause
The agent treats deprecated/ as a recycle bin instead of an audited quarantine. The "fix the error fast" impulse overrides the required analysis.

### Solution
Mandatory process before deprecating — (1) read the full file, (2) search for active references with grep, (3) verify functional duplication, (4) document the decision in DEPRECATION_LOG, (5) THEN run git mv.

### Corrective Action / Prevention
Deprecation requires prior analysis: moving code into a deprecated area is blocked unless a justifying log entry exists, and coverage of that log is verified.


### ❌ Example of the vice (Bad)
```bash
# Moves to deprecated/ without reading, without searching references, without justifying.
git mv module.py deprecated/module.py
```

### ✅ Corrected version (Good)
```bash
# Audited process before deprecating.
grep -rn "import module" src/        # 1) active references?
# 2) read the file  3) verify duplication  4) record the decision:
echo "module.py: no refs, covered by core.py - 2026-06-05" >> DEPRECATION_LOG.md
git mv module.py deprecated/module.py
```

### 🔎 Concrete detection
Pre-edit guard: reject writes/moves into deprecated/ without a corresponding
prior entry in DEPRECATION_LOG.md.
