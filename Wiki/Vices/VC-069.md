### Symptom
> Retrieval returns irrelevant, obsolete (stale index), or non-semantically cut fragments, and the agent builds on the wrong context.


### Cause
Embeddings index out of date with respect to the code, non-semantic chunking, or similarity retrieval without verifying relevance.


### Solution
Reindex on changes, chunk by semantic/symbol boundaries, and verify the fragment's relevance before using it.


### Corrective Action / Prevention
Documented; enforcement downstream. Mitigation: symbol-level retrieval (serena) or code-search MCP (claude-context) (PR-112).


### ❌ Example of the vice (Bad)
```python
# Stale index + blind chunk: retrieves an old version of the function.
ctx = index.search("refund logic")    # the index was not rebuilt after the last commit
```

### ✅ Corrected version (Good)
```python
# Reindex on changes and retrieve by symbol boundaries, verifying relevance.
if index.stale(): index.rebuild()
ctx = index.search_symbol("refund", verify_relevance=True)
```

### 🔎 Concrete detection
Retrieval index older than the code's last commit; chunks that cross function/class boundaries; a retrieved fragment used without verifying relevance. No self-contained detector (needs index/repo); see PR-047 and PR-071.

### 📚 External evidence
- **oraios/serena (verified)** — it provides semantic symbol-level retrieval precisely because raw-text retrieval feeds wrong context.
- **zilliztech/claude-context (verified)** — code-search MCP so that the retrieved context is correct and up to date.
