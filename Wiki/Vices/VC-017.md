### Síntoma (Symptom)
> Declaraciones de éxito basadas únicamente en plausibilidad estética

### Causa (Cause)
Falta de validación empírica en tiempo de ejecución o persistencia

### Solución (Solution)
Subordinar la aprobación a trazas lógicas reales o deltas observables de almacenamiento

### Acción Correctiva / Prevención
Anti-triunfalismo: el test falla si no hay evidencia JSON en .protocol/evidence/, forzando prueba empírica.

### ❌ Ejemplo del vicio (Bad)
```python
# 'Ya quedo arreglado' afirmado en el chat sin correr nada.
# El codigo se entrega con un comentario triunfalista en vez de una prueba.
def parse(x):
    return int(x)  # "fix: ahora maneja todos los casos"
```

### ✅ Versión corregida (Good)
```python
# El arreglo se respalda con un test que ejercita el caso reclamado.
def parse(x):
    return int(x.strip())

def test_parse_trims():
    assert parse("  42 ") == 42  # evidencia, no narrativa
```

### 🔎 Detección concreta
Heuristica de revision: rechazar PRs cuyo unico soporte de 'funciona' sea prosa
(comentarios/commit message) sin test nuevo que cubra el caso afirmado.
