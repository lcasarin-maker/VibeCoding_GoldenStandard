### Symptom
> Several agents operate on the same state/repo without a coordination protocol, producing race conditions, lost handoffs, or duplicated work.


### Cause
Absence of per-area ownership, of locking/quarantine, and of a handoff protocol between agents.


### Solution
Explicit protocol: per-area ownership, locks or quarantine, structured handoff, and state reconciliation.


### Corrective Action / Prevention
Documented; enforcement downstream. Recipe: declare per-area ownership, serialize access to shared state, and require structured handoff.


### ❌ Example of the vice (Bad)
```python
# Two agents edit the same file in parallel without coordination.
agent_a.edit("config.py"); agent_b.edit("config.py")   # lost update
```

### ✅ Corrected version (Good)
```python
# Ownership + lock + structured handoff.
with lock("config.py", owner="agent_a"):
    agent_a.edit("config.py")
handoff(to="agent_b", state=..., next=...)
```

### 🔎 Concrete detection
Multiple agents writing the same state without a declared lock/ownership/handoff. Extends PR-015 (dual-session drift) to N agents; see PR-040 and PR-011.
