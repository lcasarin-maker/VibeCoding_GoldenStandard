### Symptom
> A flat, readable if/elif/elif/.../else dispatch is flagged by a nesting-depth static check as deeply nested, even though no human reviewer would call it that.

### Cause
Python's grammar desugars an if/elif chain into nested ast.If nodes (each elif is an If inside the previous one's orelse), so a naive AST-depth counter measures an N-branch chain as N levels of nesting, conflating flat alternatives with genuine control-flow-inside-control-flow.

### Solution
Replace the elif chain with a dispatch dict (name -> callable); same behavior, ast.If nesting collapses from N to 1, and the metric now agrees with what a human reader already sees.

### Corrective Action / Prevention
When a nesting/complexity static check fires on an if/elif chain, convert it to a dispatch dict instead of silencing the check or manually nesting helper functions — a mechanical, behavior-preserving fix that satisfies the metric and genuinely simplifies the code.


### ❌ Example of the vice (Bad)
```python
def call_tool(self, name, arguments):
    if name == "check":
        ...
    elif name == "sync":
        ...
    elif name == "hygiene":
        ...
    elif name == "knowledge-query":
        ...
    elif name == "adn-coverage":
        ...
    elif name == "graph-context":
        ...
    elif name == "test-impact":
        ...
    else:
        raise KeyError(name)
    # ast.If nesting depth = 7 (one per elif) — a nesting gate set at >4 fires here.
```

### ✅ Corrected version (Good)
```python
def call_tool(self, name, arguments):
    dispatch = {
        "check": lambda: self._capture(self.client.command_check, ...),
        "sync": lambda: self._capture(self.client.command_sync, ...),
        "hygiene": lambda: self._capture(self.client.command_hygiene, ...),
    }
    if name not in dispatch:
        raise KeyError(name)
    return dispatch[name]()
    # ast.If nesting depth = 1.
```

### 🔎 Concrete detection
AST walk counting nested ast.If/For/While/With/Try/ExceptHandler depth; an elif chain of N branches reports depth N. Detected in CoderCerberus's own scripts/Full_dimensions_audit.py::_max_ast_nesting (D16 Legibility gate).
