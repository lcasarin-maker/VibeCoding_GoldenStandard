### Síntoma (Symptom)
> Pérdida de commits y regresiones por sobrescritura paralela

### Causa (Cause)
Modelos mentales divergentes al omitir la sincronización previa del filesystem

### Solución (Solution)
Exigir paso obligatorio de reconciliación física contra el ledger antes de escribir

### Acción Correctiva / Prevención
Dual-session drift: detectado por sync_binding --check (falla si el protocolo divergió entre sesiones).

### ❌ Ejemplo del vicio (Bad)
```python
# Dos sesiones/agentes escriben el mismo estado sin coordinacion.
# La ultima escritura gana y se pierde el trabajo de la otra (lost update).
state = load_state()
state["x"] = compute()
save_state(state)  # sin verificar version: pisa cambios concurrentes
```

### ✅ Versión corregida (Good)
```python
# Optimistic locking: la escritura falla si la version cambio entre medias.
state = load_state()
base_version = state["version"]
state["x"] = compute()
save_state_if_unchanged(state, expected_version=base_version)
```

### 🔎 Detección concreta
Detectar lectura->modificacion->escritura sobre un estado compartido sin token de
version, lock, o CAS (compare-and-swap) intermedio.
