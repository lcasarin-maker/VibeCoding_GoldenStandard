### Síntoma (Symptom)
> Se agregan riesgos

### Causa (Cause)
Supply chain no controlada

### Solución (Solution)
Gatekeeping de dependencias

### Acción Correctiva / Prevención
Ghost import (dependencia usada pero no declarada) detectado por análisis AST de imports vs manifiesto.

### ❌ Ejemplo del vicio (Bad)
```python
# El agente "alucina" una dependencia y la importa sin declararla.
# El módulo importa, pasa el linter local del agente, y revienta en CI/prod.
import requests          # NO está en requirements.txt / pyproject.toml
from rich import print   # idem

def fetch(url):
    return requests.get(url, timeout=5).json()
```

### ✅ Versión corregida (Good)
```python
# Toda dependencia usada queda declarada y fijada antes de usarse.
# pyproject.toml:
#   dependencies = ["requests==2.32.3", "rich==13.9.4"]
import requests

def fetch(url):
    return requests.get(url, timeout=5).json()
```

### 🔎 Detección concreta
Comparar el set de módulos importados (AST `import`/`importfrom`) contra el set
declarado en el manifiesto. Un import de un paquete de terceros ausente del
manifiesto es un ghost import. Herramienta de referencia: `deptry` (ver [[Project_Insights/PI-001|PI-001]]).
