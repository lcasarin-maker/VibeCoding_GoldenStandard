### Symptom
> Risks are added

### Cause
Uncontrolled supply chain

### Solution
Dependency gatekeeping

### Corrective Action / Prevention
Ghost import (dependency used but not declared) detected by AST analysis of imports vs manifest.

### ❌ Example of the vice (Bad)
```python
# The agent "hallucinates" a dependency and imports it without declaring it.
# The module imports, passes the agent's local linter, and blows up in CI/prod.
import requests          # NOT in requirements.txt / pyproject.toml
from rich import print   # same

def fetch(url):
    return requests.get(url, timeout=5).json()
```

### ✅ Corrected version (Good)
```python
# Every used dependency is declared and pinned before being used.
# pyproject.toml:
#   dependencies = ["requests==2.32.3", "rich==13.9.4"]
import requests

def fetch(url):
    return requests.get(url, timeout=5).json()
```

### 🔎 Concrete detection
Compare the set of imported modules (AST `import`/`importfrom`) against the set
declared in the manifest. An import of a third-party package absent from the
manifest is a ghost import. Reference tool: `deptry` (see [[Project_Insights/PI-001|PI-001]]).

### 📚 External evidence
- **arxiv:2603.28592** — ghost imports/dependencies are among the recurring anti-patterns of AI-generated code, present in 80-100% of the analyzed projects.
