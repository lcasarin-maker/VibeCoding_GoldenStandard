### Symptom
> The ignore list has no test verifying that each exclusion is still valid

### Cause
Silent drift: what was ignored may stop being harmless; nobody detects it

### Solution
A periodic test that checks each hard_excludes item is in the approved, documented set

### Corrective Action / Prevention
Periodically revalidate each exclusion so the ignore list cannot accumulate stale, unjustified entries.


### ❌ Example of the vice (Bad)
```ini
# A coverage exclusion nobody revalidates: hides untested code forever.
[coverage:run]
omit = src/payments/*   # added 'temporarily' 8 months ago
```

### ✅ Corrected version (Good)
```ini
# Each exclusion carries a reason and is reviewed; payments MUST be covered.
[coverage:run]
omit = src/generated/*  # only auto-generated code, justified
```

### 🔎 Concrete detection
Audit coverage 'omit'/'exclude_lines' entries without justification or review
date; especially over domain code.
