### Síntoma (Symptom)
> Lista de ignorados no tiene test que verifique que cada exclusión sigue siendo válida

### Causa (Cause)
Deriva silenciosa: lo que se ignoró puede dejar de ser inofensivo; nadie lo detecta

### Solución (Solution)
Test periódico que comprueba que cada ítem de hard_excludes está en el conjunto aprobado y documentado

### Acción Correctiva / Prevención
Checked by setup_validate.py which runs comprehensive pre-flight verification of Python, git hooks, write access, encoding, and the project registry.

### ❌ Ejemplo del vicio (Bad)
```ini
# Exclusion de cobertura que nadie revalida: oculta codigo sin testear para siempre.
[coverage:run]
omit = src/payments/*   # anadido 'temporalmente' hace 8 meses
```

### ✅ Versión corregida (Good)
```ini
# Cada exclusion lleva motivo y se revisa; payments SI debe estar cubierto.
[coverage:run]
omit = src/generated/*  # solo codigo autogenerado, justificado
```

### 🔎 Detección concreta
Auditar entradas 'omit'/'exclude_lines' de cobertura sin justificacion ni fecha de
revision; especialmente sobre codigo de dominio.
