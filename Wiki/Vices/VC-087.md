### Symptom
> Tests, scripts, or hooks leave untracked temp, probe, or backup files inside the working tree; the integrity gate flags them as zombie corruption, producing recurring fake-RED or transient-GREEN gate flips that agents "fix" by deleting the files.

### Cause
Scratch is created at repo-relative locations (TemporaryDirectory(dir=<repo>), mkdtemp(dir=<repo>), or probe/backup files written into tracked dirs) and/or cleanup is not guaranteed when a run is interrupted, so residue survives inside the tracked tree.

### Solution
Scratch must live in the OS temp dir (no dir=<repo>); cleanup runs in finally or a context manager; any unavoidable residue is gitignored. The full suite and tooling must leave `git status --porcelain` empty.

### Corrective Action / Prevention
No scratch in the tracked tree: a test or script that creates temp/probe/backup artifacts at a repo-relative path, or whose run leaves untracked non-ignored files, is rejected. Scratch goes to OS temp with guaranteed cleanup; unavoidable residue is gitignored.

### ❌ Example of the vice (Bad)
```python
with tempfile.TemporaryDirectory(dir=ROOT) as d: ...  # leaks tmpXXXX/ into the repo on interrupted runs
```

### ✅ Corrected version (Good)
```python
with tempfile.TemporaryDirectory() as d: ...  # scratch in OS temp, guaranteed cleanup
```

### 🔎 Concrete detection
Source under tests/ or scripts/ contains TemporaryDirectory(dir=<repo-root>), mkdtemp(dir=<repo-root>), or tempfile.tempdir set to a repo path; or running the suite leaves new untracked non-ignored files. Verified by test_no_scratch_in_tree in CC.
