### Symptom
> UI generated without an explicit layout

### Cause
Lack of a perceptual model

### Solution
Declare the visual structure before executing

### Corrective Action / Prevention
UI code is generated without a structured layout system, producing a fragile
visual hierarchy that breaks on different screen sizes and is unmaintainable.


### ❌ Example of the vice (Bad)
```html
<!-- Inline styles everywhere; no layout system. -->
<div style="position:absolute; top:10px; left:10px;">Header</div>
<div style="position:absolute; top:50px; left:10px;">Content</div>
```

### ✅ Corrected version (Good)
```html
<!-- Explicit layout system (CSS Grid) defines the visual structure. -->
<style>
  .container { display: grid; grid-template-rows: auto 1fr; }
</style>
<div class="container">
  <header>Header</header>
  <main>Content</main>
</div>
```

### 🔎 Concrete detection
Regex: absence of a layout framework (no CSS Grid, no Flexbox, no layout class
system) combined with presence of inline `style=` attributes on structural elements.

### 📚 External evidence
- **[[Evidence/mdn_web_docs_css_grid_layout|MDN Web Docs: CSS Grid Layout]]** — Grid and Flexbox provide explicit layout systems that replace absolute positioning.
