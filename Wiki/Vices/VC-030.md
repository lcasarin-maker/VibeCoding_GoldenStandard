### Symptom
> Risks are added

### Cause
Uncontrolled supply chain

### Solution
Dependency gatekeeping

### Corrective Action / Prevention
Every imported dependency must be declared in the manifest: a check fails when code imports a package that the dependency manifest does not list.


### ❌ Example of the vice (Bad)
```text
# requirements.txt
requests==2.32.3

# app.py
import yaml
data = yaml.safe_load("name: Cerberus")
```

### ✅ Corrected version (Good)
```text
# requirements.txt
PyYAML==6.0.2
requests==2.32.3

# app.py
import yaml
data = yaml.safe_load("name: Cerberus")
```

### 🔎 Concrete detection
Parse the top-level imports and compare them with the dependency manifest (requirements/lock/pyproject metadata); fail when an import resolves to a package that is absent from the manifest.

### 📚 External evidence
- **generate_golden_audit.py** — imports yaml to parse the catalogs, so PyYAML is a real runtime dependency of the audit generator.
- **scripts/validate_golden_standard_catalogs.py** — imports yaml for validation, so the manifest gate must cover that dependency too.
- **scripts/metrics.py** — imports yaml for metrics generation, so an undeclared PyYAML dependency would break the stubs gate.
