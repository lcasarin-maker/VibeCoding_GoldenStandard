### Symptom
> Directory added to the ignore list without verifying its content

### Cause
Implicit trust that the invisible is harmless

### Solution
Audit content before excluding; document the justification with a date; move to deprecated if it has vices

### Corrective Action / Prevention
Physical prevention by verifying that every active exclusion rule in the .gitignore file is preceded by an explanatory comment justifying the exclusion in the same block.

### ❌ Example of the vice (Bad)
```text
# .gitignore: a directory is ignored without knowing what it contains.
scratch/
```

### ✅ Corrected version (Good)
```text
# Each exclusion carries a reason and date; it was audited before ignoring.
scratch/   # build artifacts only, audited 2026-06-05, no secrets
```

### 🔎 Concrete detection
.gitignore entries / coverage exclusions without a justifying comment,
especially opaque, unaudited directories.
