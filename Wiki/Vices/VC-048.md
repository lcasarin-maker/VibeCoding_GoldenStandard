### Symptom
> All context is mixed together

### Cause
Lack of stratification

### Solution
Separate core, active, patterns, and history

### Corrective Action / Prevention
validate_gs_topology.py checks that GS exposes separate canonical surfaces for Vices, Tokenomics, Project Insights, and the graph instead of one monolithic knowledge page.

### ❌ Example of the vice (Bad)
```text
# A single CONTEXT.md mixes core, active state, patterns, and history.
```

### ✅ Corrected version (Good)
```text
# Separated by lifespan: CORE.md (immutable), STATUS.md (active), PATTERNS.md, HISTORY.md.
```

### 🔎 Concrete detection
A single memory file that mixes layers with different lifespans (stable vs volatile).
