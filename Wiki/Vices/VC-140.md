# VC-140: Continuity gap between agents (orphan handoff)

| Field | Detail |
|---|---|
| **ID** | `VC-140` |
| **Category** | Vibe Coding |
| **Status** | **PREVENTED** |
| **Severity** | **high** |
| **Depth** | 🟢 Deep |
| **Tags** | `continuity`, `agent-agnostic`, `prevented-mechanically` |
| **Downstream Verification** | `none` |
| **Validation Mechanism** | `check_handoff_freshness` |

---

### Symptom
> The next agent re-derives the context from scratch, spends tokens, or strays from the task.

### Cause
Changes committed without an explicit, fresh handoff for the next agent.

### Solution
A living HANDOFF.md with a fixed schema, mandatory and mechanically verified on every commit (agent-agnostic).

### Corrective Action / Prevention
Physical verification: scripts/check_handoff_freshness.py invoked in the git commit-msg hook
(agnostic: runs with any `git commit`, whether Codex/Gemini/Claude). Blocks the commit if
there are substantive changes but HANDOFF.md was not updated or is missing schema sections
(ESTADO/SIGUIENTE/VERIFICAR). Explicit escape: the [skip-handoff] token in the message or
env CERBERUS_SKIP_HANDOFF=1.


### ❌ Example of the vice (Bad)
```markdown
# commit with new code and HANDOFF.md untouched -> the next agent loses the thread
git commit -m "fix: parser"
```

### ✅ Corrected version (Good)
```markdown
# HANDOFF.md updated and staged in the same commit, with the fixed schema
## ESTADO   parser fixed, suite green
## SIGUIENTE 1. migrate the lexer in scripts/lexer.py
## VERIFICAR pytest -q
```

### 🔎 Concrete detection
Substantive changes staged without a fresh HANDOFF.md in the changeset, or HANDOFF.md missing
the mandatory sections. Mechanically verified by check_handoff_freshness in commit-msg.

### 📚 External evidence
- **CoderCerberus v0.5 -- multi-agent operation (Codex/Gemini/Claude)** — Context loss between agent handoffs wastes tokens and causes drift; a mandatory fresh handoff prevents it.
