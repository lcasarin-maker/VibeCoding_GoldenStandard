# VC-140: Brecha de continuidad entre agentes (handoff huerfano)

| Campo | Detalle |
|---|---|
| **ID** | `VC-140` |
| **Categoría** | Vibe Coding |
| **Estado** | **PREVENTED** |
| **Severidad** | **high** |
| **Profundidad** | 🟢 Deep |
| **Tags** | `continuity`, `agent-agnostic`, `prevented-mechanically` |
| **Downstream Verification** | `none` |
| **Mecanismo de Validación** | `check_handoff_freshness` |

---

### Síntoma (Symptom)
> El siguiente agente re-deriva el contexto desde cero, gasta tokens o se desvia de la tarea.

### Causa (Cause)
Cambios commiteados sin un relevo (handoff) explicito y fresco para el proximo agente.

### Solución (Solution)
HANDOFF.md vivo de esquema fijo, obligatorio y verificado mecanicamente en cada commit (agente-agnostico).

### Acción Correctiva / Prevención
Verificacion fisica: scripts/check_handoff_freshness.py invocado en el git hook commit-msg
(agnostico: corre con cualquier `git commit`, sea Codex/Gemini/Claude). Bloquea el commit si
hay cambios sustantivos pero HANDOFF.md no se actualizo o le faltan secciones del esquema
(ESTADO/SIGUIENTE/VERIFICAR). Escape explicito: token [skip-handoff] en el mensaje o
env CERBERUS_SKIP_HANDOFF=1.


### ❌ Ejemplo del vicio (Bad)
```markdown
# commit con codigo nuevo y HANDOFF.md sin tocar -> el proximo agente pierde el hilo
git commit -m "fix: parser"
```

### ✅ Versión corregida (Good)
```markdown
# HANDOFF.md actualizado y staged en el mismo commit, con el esquema fijo
## ESTADO   parser arreglado, suite verde
## SIGUIENTE 1. migrar el lexer en scripts/lexer.py
## VERIFICAR pytest -q
```

### 🔎 Detección concreta
Cambios sustantivos staged sin HANDOFF.md fresco en el changeset, o HANDOFF.md sin las
secciones obligatorias. Verificado mecanicamente por check_handoff_freshness en commit-msg.

### 📚 Evidencia externa
- **CoderCerberus v0.5 -- operacion multi-agente (Codex/Gemini/Claude)** — La perdida de contexto entre relevos de agente desperdicia tokens y causa desviacion; un handoff fresco obligatorio lo previene.
