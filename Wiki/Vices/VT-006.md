### Síntoma (Symptom)
> Corre sin verificar

### Causa (Cause)
Oráculo ausente

### Solución (Solution)
Aserción obligatoria

### Acción Correctiva / Prevención
Checked by D9 (Test Purity) using AST TestTheaterVisitor to flag assert True, assertTrue(True), assertEqual(x, x), and tests without active asserts.

### ❌ Ejemplo del vicio (Bad)
```python
# El agente genera un test que ejecuta el código pero no verifica nada.
# Pasa siempre (incluso si discount() devuelve basura) -> teatro de seguridad.
def test_apply_discount():
    cart = Cart(items=[Item(price=100)])
    cart.apply_discount(0.1)
    # sin assert: el método se ejecuta y el test "pasa"
```

### ✅ Versión corregida (Good)
```python
# El test fija un oráculo explícito sobre el resultado observable.
def test_apply_discount():
    cart = Cart(items=[Item(price=100)])
    cart.apply_discount(0.1)
    assert cart.total() == 90  # comportamiento, no mera ejecución
```

### 🔎 Detección concreta
AST: marcar funciones `test_*` cuyo cuerpo no contenga ningún `assert` activo, o
cuyos asserts sean tautológicos (`assert True`, `assertEqual(x, x)`). Un test sin
oráculo no puede fallar ante una regresión.
