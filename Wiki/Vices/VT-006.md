### Symptom
> Runs without verifying

### Cause
Missing oracle

### Solution
Mandatory assertion

### Corrective Action / Prevention
A test that runs without any assertion verifies nothing; at least one discriminating assertion is mandatory.


### ❌ Example of the vice (Bad)
```python
# The agent generates a test that runs the code but verifies nothing.
# Always passes (even if discount() returns garbage) -> security theater.
def test_apply_discount():
    cart = Cart(items=[Item(price=100)])
    cart.apply_discount(0.1)
    # no assert: the method runs and the test "passes"
```

### ✅ Corrected version (Good)
```python
# The test fixes an explicit oracle on the observable result.
def test_apply_discount():
    cart = Cart(items=[Item(price=100)])
    cart.apply_discount(0.1)
    assert cart.total() == 90  # behavior, not mere execution
```

### 🔎 Concrete detection
AST: flag `test_*` functions whose body contains no active `assert`, or
whose asserts are tautological (`assert True`, `assertEqual(x, x)`). A test without
an oracle cannot fail on a regression.

### 📚 External evidence
- **arxiv:2602.00409** — coding agents generate tests without assertions and without invoking the focal method at a measurable frequency; excessive mocking aggravates the problem.
