### Symptom
> A tool call fails (timeout, error, unexpected output) and the agent continues as if it had succeeded, or enters a retry loop without backoff.


### Cause
The result of each tool call is not validated; absence of error handling and of a backoff/limit policy in the agent loop.


### Solution
Validate each tool result; apply backoff and a retry limit; degrade explicitly on persistent failure.


### Corrective Action / Prevention
Static detector `vc065_unhandled_tool_call_failure` flags tool calls whose result is consumed without verifying success.

### ❌ Example of the vice (Bad)
```python
result = call_tool("read_file", path)   # returned an error
proceed(result)                          # the error is used as if it were content
```

### ✅ Corrected version (Good)
```python
result = call_tool("read_file", path)
if not result.ok:
    result = retry_with_backoff(max=3) or degrade()   # backoff + limit + degradation
```

### 🔎 Concrete detection
Tool-call results used without verifying success; retry loops without backoff or limit. Related to PR-066 (runaway loops) and VC-017 (no timeout).
