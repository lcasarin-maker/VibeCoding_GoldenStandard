# CD18 — Multi-Agent Project Governance

Defines the structural invariants that keep multi-agent projects continuous, authoritative, and free of broken bridges: single governance hub, mandatory session close, live-only backlog, resolved debt in git only, consistent federated structure, and explicit task-completion authority.

---

## Graph Role

Structural foundation hub: highest fan-out in GS, touching CD01, CD02, CD04, CD05, CD07, CD13, CD14, CD15, CD16. CD13 (Knowledge Canonicalization) is the conceptual origin — GS's ingestion discipline inspired the session-close pattern here. CD01 is the most-loaded neighbor (4 SP rules); changes to either domain should trigger review of the other.

---

## Covers

- Single governance hub — all agent rules in one file
- Mandatory session-close discipline (HANDOFF + STATE + audit/sessions)
- Live-debt-only backlog; resolved debt to git log
- Consistent folder structure enforced across federated repos
- Template compliance for tasks and session records
- Agent self-certification of task completion against DoD

---

## Explicitly Excludes

- Per-vendor agent capabilities or brand-specific tooling
- Code quality vices inside files (covered by VC/VT catalogs)
- Token spend and context efficiency (covered by CD11)

### Linked Principles

*   **[[Principles/SP-001|SP-001]]** (Project: *canonical-structure*): Collapse all agent governance into .agents/AGENTS.md; delete all other governance files.
*   **[[Principles/SP-002|SP-002]]** (Project: *canonical-structure*): Write every material fact into project files before session ends; chat is not durable storage.
*   **[[Principles/SP-003|SP-003]]** (Project: *canonical-structure*): Keep tasks/backlog/ live-only; completed items are deleted after their closing commit.
*   **[[Principles/SP-004|SP-004]]** (Project: *canonical-structure*): Delete task files on completion; the git commit message is the sole durable record.
*   **[[Principles/SP-005|SP-005]]** (Project: *canonical-structure*): Session close (HANDOFF + STATE + audit/sessions record) is a required phase, not optional.
*   **[[Principles/SP-006|SP-006]]** (Project: *canonical-structure*): audit.py enforces the canonical tree across CC, GS, and satellites; failing audit blocks session close.
*   **[[Principles/SP-007|SP-007]]** (Project: *canonical-structure*): tasks/README.md and audit/README.md templates are mandatory; agents must use them.
*   **[[Principles/SP-008|SP-008]]** (Project: *canonical-structure*): Agent self-certifies done against AGENTS.md §4 DoD; review/ only for explicitly flagged tasks.
*   **[[Principles/SP-009|SP-009]]** (Project: *canonical-structure*): Canonicalize every material decision, session finding, and new work item before session ends — same gate as GS INGESTION_PROTOCOL, applied to project knowledge.
