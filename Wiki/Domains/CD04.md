# CD04 — Code Vitality & Dead Surface

Distinguishes active, load-bearing logic from inert, spectral, or ceremonial code surface so the repository does not accumulate dead operational mass.

---

## Graph Role

Candidate-orphan pressure valve: this domain questions whether a file exists because it is alive or because nobody removed it.

---

## Covers

- Dead or spectral executable surface
- Residue that pretends to be active logic
- Helpers that survive after their real load-bearing role is gone

---

## Explicitly Excludes

- Tree-level ghost files as a repository-integrity problem
- Coverage adequacy
- Runtime drift across repos

### Linked Principles

*   **[[Principles|PR-085]]** (Project: *reference*): Treat spectral scripts, stale helper layers, and dead refactor residue as a governance failure, not as harmless leftovers.
*   **[[Principles|PR-097]]** (Project: *reference*): Temporary helpers must disappear unless they remain reusable, documented, and visibly load-bearing.
