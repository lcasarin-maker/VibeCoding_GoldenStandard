# CD11 — Context Efficiency & Tokenomics

Treats token spend, context hygiene, trimming, retrieval, and memory discipline as operational efficiency concerns subordinate to code quality.

- Context hygiene, trimming, and memory discipline
- Tooling that reduces rereads and noisy interaction

### Linked Principles

*   **[[Principles|PR-081]]** (Project: *tokencost*): Meter tokens before and during LLM calls so spend is visible before usage grows.
*   **[[Principles|PR-083]]** (Project: *litellm*): Unify provider routing, fallback, and cost tracking so efficiency and resilience are handled once.
*   **[[Principles|PR-085]]** (Project: *reference*): Treat output trimming, context hygiene, and history externalization as first-class controls.
*   **[[Principles|PR-109]]** (Project: *ratelimit*): Use rate limiting to avoid quota surprises and reduce reactive token waste.
*   **[[Principles|PR-112]]** (Project: *serena*): Prefer symbol-level retrieval so recovered context is correct instead of bloated by blind chunking.
