Este dominio cubre un área específica del protocolo de consistencia y seguridad del GS. Los siguientes insights satélite están vinculados a esta lente operativa:

### Insights Vinculados

*   **[[Project_Insights/PI-002|PI-002]]** (Proyecto: *pytest-good-assertions*): Keep tests high-signal: assertions should discriminate behavior, not merely confirm presence.
*   **[[Project_Insights/PI-001|PI-001]]** (Proyecto: *deptry*): Prevent dependency drift from destabilizing the test suite by validating imports before running coverage gates.
*   **[[Project_Insights/PI-012|PI-012]]** (Proyecto: *cerberus*): Reject fake coverage patterns such as xfail-as-expected, placeholder tests, mocks without intent or broad skips without evidence.
*   **[[Project_Insights/PI-015|PI-015]]** (Proyecto: *cerberus*): Require each new guard to break a real circularity and reduce the baseline instead of merely naming the problem.
*   **[[Project_Insights/PI-017|PI-017]]** (Proyecto: *cerberus*): Split broad coverage theater into discriminative checks so one test never pretends to cover many unrelated vices.
