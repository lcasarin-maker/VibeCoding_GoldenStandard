# CD01 — Repository Integrity & Surface Hygiene

Controls the active repository surface so ghost artifacts, stale residue, and polluted roots do not masquerade as live knowledge or valid output.

---

## Graph Role

High-value graph hub for active-surface discoverability: if this domain drifts, the graph loses trust in what is live versus archival.

---

## Covers

- Ghost or undeclared active artifacts
- Root hygiene after audits and refactors
- Archive boundaries between active and historical surfaces

---

## Explicitly Excludes

- Dead code semantics inside otherwise active files
- Dependency CVEs or supply-chain freshness
- Token spend and context cost

---

### Linked Principles

*   **[[Principles/PR-085|PR-085]]** (Project: *reference*): Treat leftover backups, stale plan docs, spectral scripts, and orphan residue as governance debt, not harmless clutter.
*   **[[Principles/PR-088|PR-088]]** (Project: *reference*): Keep the root workspace clean after every audit or remediation pass so historical evidence never pollutes active context.
*   **[[Principles/PR-097|PR-097]]** (Project: *reference*): Prefer simple, auditable execution surfaces and remove temporary helpers unless they are reusable and documented.
